## 2. v0.1.4 リリース概要
`v0.1.4` は、`v0.1.3` 以降に積み上げた UI 改善、OCR 安定化、設定整理、プロジェクト保存、性能改善をまとめて反映したパッチリリースである。

主な更新点:
- メイン画面サイズ復元、初期サイズ調整、タイムライン右ペイン最小幅調整、処理状況ペイン高さ調整
- 設定画面サイズ見直し、利用者向けラベル整理、GPU / worker 数設定、ライセンス / バージョン情報導線追加
- プレビュー別画面ポップアップと、メイン画面への復帰動作追加
- `frames.csv` / `segments.csv` の Excel 文字化け対策
- `font_size` 列追加、複数テロップ同時強調改善、文字色推定改善
- `movie-telop-transcriber.settings.json` への利用者設定統合
- `.mtproj` プロジェクト保存 / 再読込
- OCR warmup、warmup 前倒し、候補フレーム選別、GPU 2 worker 評価と設定化
- benchmark sample suite と性能ログ整備

## 3. v0.1.4 配布物
GitHub Release `v0.1.4` では、次の asset を公開対象とする。

```text
movie-telop-transcriber-win-x64-v0.1.4.zip
movie-telop-transcriber-win-x64-v0.1.4.zip.sha256
Install-MovieTelopTranscriber.ps1
Install-MovieTelopTranscriber.ps1.sha256
```

アプリ本体 zip には、WinUI 3 アプリの Release build 一式、`tools/ocr/paddle_ocr_worker.py`、インストーラ、アンインストーラ、利用者向け文書、最小サンプルを含める。

Python runtime、PaddlePaddle / PaddleOCR の Python package、OCR モデル本体は zip に含めない。インストーラまたは導入手順に沿って別途導入する。

## 4. v0.1.4 導入前提
- 対象 OS は Windows x64 とする。
- アプリの TargetPlatformMinVersion は `10.0.17763.0` である。
- Release build 出力の `.runtimeconfig.json` により、.NET 10 Desktop Runtime x64 を前提にする。
- 実動画 OCR は、Python `3.10.x`、PaddlePaddle `3.2.0` CPU または GPU 対応 runtime、PaddleOCR `3.5.0`、`PP-OCRv5_server_det` / `PP-OCRv5_server_rec` を前提にする。
- 導入手順は `docs/12_導入手順書.md` を参照する。

## 5. v0.1.4 の既知制約

### 5.1 配布物
- `dotnet publish --self-contained true` の出力は、過去検証で `0xC000027B` により起動失敗したため、現時点では使わない。
- 配布 zip の基準は Release build 出力であり、サイズは大きい。

### 5.2 インストーラ
- インストーラのオンライン導入では、GitHub Release、PaddlePaddle wheel 配布、PyPI、PaddleOCR モデル配布へ接続できる必要がある。
- Python 3.10 は利用者に導入してもらう前提である。既定では `py -3.10` を使い、導入後は `movie-telop-transcriber.settings.json` に解決結果を書き戻す。
- プロキシや対話型ネットワーク制約がある端末では、オフライン導入手順で OCR runtime と `.paddlex` を別配布する。

### 5.3 OCR
- 配布 zip 内の `app\movie-telop-transcriber.settings.json` は、`..\ocr-runtime\.venv\Scripts\python.exe` を既定の OCR Python として参照する。`app` フォルダだけを単独移動すると、この既定パスは成立しない。
- PaddleOCR 用 Python が未導入、または `paddleocr` package が入っていない場合、実動画 OCR は失敗する。
- CPU 推論は動画長、抽出間隔、出力列、モデル初期化状況によって時間が大きく変わる。
- GPU 2 worker は RTX 2070 SUPER 検証で短縮を確認したが、既定値は `1` のままとする。

### 5.4 テロップ属性
- `font_size` は OCR 矩形高さベースの推定値であり、実フォントサイズそのものではない。
- `text_color`、`stroke_color`、`text_type` は属性解析結果であり、背景色や特殊装飾により誤差が残る場合がある。
- `font_family` と `background_color` は `null` を返す。
- `segments.ass` はスタイル `Default` のみを出力する。

### 5.5 タイムライン / プレビュー
- タイムライン上の結合 / 分割 UI は実装済みだが、今後の再設計余地は残る。
- プレビュー別画面化は利用可能だが、運用上はメイン画面との同期を前提にする。
