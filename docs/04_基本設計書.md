# 基本設計書

## 1. 文書情報
- システム名: movie-telop-transcriber
- 文書種別: 基本設計書
- 作成日: 2026-04-28
- 対象工程: 設計

## 2. 目的
仕様書で確定した要件と仕様を、実装可能なシステム構成へ落とし込む。  
本書では、アプリ全体の責務分離、主要モジュール、データの受け渡し、ファイル配置の大枠を定義する。

## 3. システム全体構成
本システムは、Windows 上で動作する GUI アプリ本体と、動画解析処理群から構成する。

主要構成要素:
1. GUI アプリ本体
2. 動画入力・メタデータ取得層
3. フレーム抽出層
4. OCR ワーカー層
5. 属性解析層
6. テロップ統合層
7. 出力層
8. ログ・作業ディレクトリ管理層

## 4. アーキテクチャ方針
### 4.1 基本方針
- GUI と解析処理を責務分離する。
- OCR エンジンへの依存はワーカー層へ閉じ込める。
- フレーム抽出、OCR、属性解析、統合、出力を段階的に再実行できる構成にする。
- JSON を正本、CSV を派生出力とする。

### 4.2 採用イメージ
- フロントエンド: WinUI 3
- アプリケーション制御: C# 側のオーケストレーション
- OCR ワーカー: ローカル実行コンポーネント
- 永続化: ローカルファイルベース

## 5. モジュール構成
### 5.1 GUI アプリ本体
責務:
- 動画選択
- 設定入力
- 実行開始 / 停止
- 進捗表示
- タイムライン表示
- 結果参照
- サブ画面管理

入力:
- 利用者操作
- 処理進捗
- 解析結果

出力:
- 実行要求
- 設定情報
- 保存要求

### 5.2 動画入力・メタデータ取得層
責務:
- 動画ファイルの存在確認
- 対応形式候補確認
- デコード可否確認
- 動画メタデータ取得

出力:
- `source_video` 情報
- 読み込み可否結果

### 5.3 フレーム抽出層
責務:
- 時刻基準でフレーム抽出を行う
- 抽出画像を作業ディレクトリへ保存する
- フレーム一覧を生成する

出力:
- フレーム画像群
- フレーム単位メタデータ

### 5.4 OCR ワーカー層
責務:
- フレーム画像から文字領域検出を行う
- OCR により文字列を取得する
- OCR 結果を共通形式へ正規化する

入出力方針:
- GUI 本体は OCR エンジン固有 API を直接知らない
- ワーカー層は共通の入出力契約を返す

### 5.5 属性解析層
責務:
- 文字色、枠色、地色を推定する
- フォント分類、サイズ推定を行う
- `text_type` の候補を付与する

出力:
- フレーム単位の属性付き検出結果

### 5.6 テロップ統合層
責務:
- 連続フレーム上の同一テロップを統合する
- セグメント単位の開始時刻、終了時刻、代表属性を確定する

出力:
- セグメント単位結果

### 5.7 出力層
責務:
- JSON 出力
- CSV 出力
- 実行ログ要約出力

### 5.8 ログ・作業ディレクトリ管理層
責務:
- 作業ディレクトリの作成
- フレーム画像、ログ、出力ファイルの配置
- 再実行時の既存成果物との整合確認

## 6. データフロー
1. GUI が動画ファイルと解析設定を受け取る。
2. 動画入力層が入力を検証し、メタデータを返す。
3. フレーム抽出層が時刻基準でフレーム画像を生成する。
4. OCR ワーカー層がフレーム単位の文字検出と文字認識を行う。
5. 属性解析層が色、フォント、サイズ、種別候補を付与する。
6. テロップ統合層がセグメント単位結果へ統合する。
7. 出力層が JSON / CSV / ログを保存する。
8. GUI が進捗と結果を表示する。

GUI 内の表示連動:
1. セグメント結果を受け取ったらタイムライン表示モデルを生成する。
2. タイムライン選択時は、該当セグメントを結果一覧とプレビューへ反映する。
3. 結果一覧選択時は、タイムライン上の該当区間をハイライトする。
4. プレビュー時刻移動時は、タイムラインカーソル位置を更新する。

## 7. データ受け渡し方針
### 7.1 層間の基本ルール
- 各層は入力と出力を明確に分ける。
- OCR エンジン固有の生データは、ワーカー層外へ漏らさない。
- フレーム単位結果とセグメント単位結果は明確に分離する。

### 7.2 主要な受け渡し単位
- `VideoSource`
  - 入力動画情報
- `FrameRecord`
  - 抽出フレーム情報
- `DetectionRecord`
  - OCR と属性解析を含むフレーム単位検出情報
- `SegmentRecord`
  - 統合済みテロップ情報
- `TimelineSegmentViewModel`
  - タイムライン表示用のセグメント情報
- `ExportPackage`
  - JSON / CSV / ログ出力対象

### 7.3 タイムライン表示モデル方針
- タイムライン表示は `SegmentRecord` から生成する。
- 表示に必要な最小属性は以下とする。
  - `segment_id`
  - `start_timestamp_ms`
  - `end_timestamp_ms`
  - `text`
  - `text_type`
  - `selected`
- GUI 表示専用の状態は永続データモデルへ混在させない。

## 8. 非機能設計の基本方針
### 8.1 オフライン性
- 主要処理はローカルで完結する。
- 外部 API 呼び出しを必須にしない。

### 8.2 再実行性
- フレーム抽出以降の工程を段階的にやり直せる余地を持たせる。
- 中間成果物を作業ディレクトリへ保存する。

### 8.3 保守性
- GUI、OCR、属性解析、統合、出力の責務を分ける。
- 将来の OCR 差し替えを可能にする。

### 8.4 ログ性
- 入力条件
- 抽出設定
- OCR エンジン名
- 処理時間
- 警告件数
- エラー件数

を最低限ログ化する。

## 9. ディレクトリ構成の基本方針
リポジトリ内の役割分担は以下を基本とする。
- `src/`
  - アプリ本体と処理実装
- `docs/`
  - 要件、仕様、設計、テスト、運用文書
- `work/`
  - 実行時作業領域
- `out/`
  - 出力成果物

※ `work/` と `out/` は実行時生成物として `.gitignore` 対象とする。

### 9.1 実行時ディレクトリ構成案
`work/` 配下:
- `work/runs/<run_id>/frames/`
  - 抽出フレーム画像
- `work/runs/<run_id>/ocr/`
  - OCR ワーカー生結果
- `work/runs/<run_id>/attributes/`
  - 属性解析結果
- `work/runs/<run_id>/logs/`
  - 実行ログ

`out/` 配下:
- `out/<run_id>/segments.json`
- `out/<run_id>/segments.csv`
- `out/<run_id>/frames.csv`
- `out/<run_id>/summary.log`

### 9.2 配置方針
- 実行ごとに `run_id` 単位でディレクトリを分ける。
- 中間成果物と最終成果物を分離する。
- JSON を正本、CSV を派生物として扱う。
- ログは中間成果物側と最終成果物側の両方から追跡できるようにする。

### 9.3 クリーンアップ方針
- `work/runs/<run_id>/` は再実行や検証のため保持可能とする。
- GUI から実行単位で削除できる構成を優先する。
- `out/<run_id>/` は成果物として明示的削除以外では保持する。

## 12. 画面設計
### 12.1 メイン画面構成
メイン画面は 3 ペイン構成とし、以下の責務を持たせる。

#### 左ペイン
目的:
- 入力と操作の集約

主な要素:
- 動画ファイル選択ボタン
- 入力動画情報表示
- 抽出間隔入力
- 開始時刻入力
- 終了時刻入力
- 出力先表示
- 解析開始ボタン
- 解析停止ボタン
- 結果保存ボタン
- 処理状態表示

#### 中央ペイン
目的:
- 現在の動画状態と時間軸状態の確認

主な要素:
- 動画プレビュー領域
- 再生位置表示
- タイムライン表示領域
- 現在フレーム情報表示
- 選択中セグメント要約表示

#### 右ペイン
目的:
- 抽出結果の確認と選択

主な要素:
- セグメント一覧
- 検索または絞り込み入力
- 選択中セグメント詳細
- 属性表示

### 12.2 タイムライン UI 構成
タイムライン領域は少なくとも以下で構成する。
- 動画全長を示す時間軸
- セグメント帯表示
- 現在再生位置カーソル
- 選択セグメントハイライト

補足:
- 初期実装では、秒単位目盛りを優先する。
- ズーム機能は設計余地として残すが、初期実装必須とはしない。

### 12.3 一覧表示方針
セグメント一覧は表形式を基本とする。

初期列候補:
- 開始時刻
- 終了時刻
- テキスト
- 種別
- 信頼度

期待動作:
- 行選択でタイムラインとプレビューを同期する。
- ソート可能な余地を持たせる。

### 12.4 サブ画面設計
サブ画面は非モーダルとし、以下を分離する。

#### 設定詳細画面
- OCR エンジン設定
- 作業ディレクトリ設定
- ログ設定

#### 出力結果詳細画面
- JSON 出力結果の詳細確認
- CSV 出力内容の確認

#### エラー詳細画面
- エラーコード
- エラーメッセージ
- 回復可能性

#### 属性詳細画面
- 色属性
- フォント属性
- 種別属性

### 12.5 主要UIコンポーネント方針
WinUI 3 前提で、以下の標準コンポーネント採用を基本とする。
- `NavigationView` は初期画面 1 枚構成のため必須としない。
- `Grid` / `SplitView`
  - 3 ペイン構成のレイアウトに使用
- `TextBox` / `NumberBox`
  - 抽出間隔や時刻入力
- `Button`
  - 実行操作
- `InfoBar`
  - 実行状態、警告、エラー表示
- `ListView` または表形式相当コンポーネント
  - セグメント一覧
- `ContentDialog` ではなく独立ウィンドウ相当の非モーダル画面
  - サブ画面用途

### 12.6 画面状態管理
GUI 状態は少なくとも以下を持つ。
- 入力未選択
- 動画読込済み
- 実行中
- 実行完了
- エラー発生

状態遷移時の基本挙動:
- 実行中は開始ボタンを無効化する。
- 実行中でも一覧参照とタイムライン確認は可能にする。
- エラー発生時は `InfoBar` とエラー詳細画面の両方で追跡できるようにする。
