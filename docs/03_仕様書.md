## 4. 主要決定事項
- Windows GUI アプリとして構成する
- メイン画面 1 画面完結を基本とする
- サブ画面は非モーダルとする
- 動画タイムライン上でセグメント単位のテロップ確認を可能にする
- OCR はローカルワーカー分離構成を前提とする
- JSON を正本、CSV を派生出力とする

## 5. 設計への引き継ぎ
- 画面構成、タイムライン表示、OCR 連携、属性形式、出力形式は個別仕様書を参照する
- 設計工程では `docs/04_基本設計書.md` と `docs/05_詳細設計書.md` を入口とする
