# 要件定義書

## 1. 文書情報
- システム名: movie-telop-transcriber
- 文書種別: 要件定義書
- 作成日: 2026-04-28
- 対象: 初版

## 2. 背景
動画内のテロップを手作業で確認し、文字列、時刻、見た目の種類を記録する作業は負荷が高い。そこで、動画を入力するとテロップ情報を抽出し、再利用可能な構造化データとして保存できる GUI アプリを開発する。

## 3. 目的
- 動画からテロップ文字列を抽出する。
- テロップごとの時刻情報を記録する。
- テロップの見た目情報を整理して記録する。
- オフライン環境でも継続利用できる業務向け GUI アプリを提供する。

## 4. 対象範囲
### 4.1 対象
- ローカル動画ファイルの読み込み
- フレーム抽出
- テロップ文字認識
- テロップ種別情報の記録
- 結果の保存と再確認

### 4.2 対象外
- クラウド OCR への依存
- 常時オンライン接続を前提とした機能
- 動画編集そのもの
- 音声文字起こし

## 5. 利用者
- 動画制作者
- 動画アーカイブ担当者
- テロップ整理や素材管理を行う運用担当者

## 6. 利用環境要件
- オフライン環境で動作すること
- GUI アプリとして操作できること
- ローカル PC 上の動画ファイルを直接扱えること

## 7. 機能要件
### 7.1 動画入力
- 利用者はローカルの動画ファイルを選択できること。
- 読み込んだ動画の基本情報を表示できること。
  - ファイル名
  - 長さ
  - 解像度
  - フレームレート

### 7.2 フレーム抽出
- 一定間隔でフレームを抽出できること。
- 抽出間隔を利用者が設定できること。
- 抽出時刻とフレーム番号を保持できること。

### 7.3 テロップ検出と文字起こし
- 抽出フレームからテロップ候補領域を扱えること。
- テロップ文字列を OCR で取得できること。
- OCR 結果の信頼度を保持できること。

### 7.4 テロップ属性記録
- 文字列ごとに時刻を記録できること。
- 次の属性を記録対象とすること。
  - フォント
  - サイズ
  - 文字色
  - 枠色
  - 地色
- 属性値が特定できない場合でも、不明として保持できること。

### 7.5 テロップ統合
- 連続フレームで同一と判断されるテロップを統合できること。
- 統合結果として、開始時刻と終了時刻、代表文字列、属性情報を保持できること。

### 7.6 結果確認
- GUI 上で抽出結果を一覧確認できること。
- 該当時刻やフレームを再確認できること。
- 必要に応じて結果を再保存できること。

### 7.7 データ出力
- 結果を JSON 形式で保存できること。
- 必要に応じて CSV 形式でも出力できること。

## 8. 非機能要件
### 8.1 オフライン動作
- 主要機能はインターネット接続なしで利用できること。
- OCR や解析処理はローカル実行可能な構成であること。

### 8.2 操作性
- GUI により、動画選択、解析開始、進捗確認、結果確認ができること。
- 専門知識がなくても主要操作が追える画面構成であること。

### 8.3 保守性
- フレーム抽出、OCR、属性解析、統合処理を分離した構成であること。
- 処理失敗時にログから原因を追跡できること。

### 8.4 性能
- 長時間動画でも段階的に処理できること。
- 全件再処理だけでなく、一部工程の再実行がしやすいこと。

## 9. 制約条件
- オフライン前提のため、クラウド API への必須依存は持たない。
- フォント推定は厳密一致ではなく、候補推定または分類になる可能性がある。
- 色推定は動画圧縮やエフェクトの影響を受ける可能性がある。

## 10. 想定入出力
### 10.1 入力
- 動画ファイル
- 抽出間隔
- 解析オプション

### 10.2 出力
- フレーム単位解析結果
- テロップ単位統合結果
- ログ
