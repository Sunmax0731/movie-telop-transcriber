# 既知不具合と制約一覧

## 1. 文書情報
- 対象 Issue: `#47`
- 作成日: 2026-04-28
- 対象工程: テスト
- 記録者: Codex

## 2. 目的
テスト工程で確認した不具合、既知制約、回避策、後続 Issue を整理し、リリース工程での判断材料にする。

## 3. サマリ
- テスト工程の主要シナリオは通過しており、アプリ本体の起動、動画入力、フレーム抽出、OCR sidecar 読み込み、属性解析、出力、ログ、段階的再実行は成立している。
- 現時点で、テスト工程を継続不能にする重大不具合は残っていない。
- リリース工程では、実 OCR worker、配布構成、導入手順、属性推定の扱いを確定する必要がある。

## 5. 既知制約
| ID | 内容 | 影響 | 回避策 | 後続 Issue | リリース判断 |
| --- | --- | --- | --- | --- | --- |
| C-001 | OCR エンジン未指定かつ外部 worker 未指定の場合は、実動画 OCR の既定として PaddleOCR を使う。`json-sidecar` は `MOVIE_TELOP_OCR_ENGINE=json-sidecar` を明示した検証モードとし、sidecar がない場合は `OCR_SIDECAR_NOT_FOUND` の OCR エラーになる。Windows OCR baseline worker は追加済みだが、日本語テロップの認識精度が不足したため PaddleOCR PP-OCRv5 worker を採用する。 | PaddleOCR 採用時は Python ランタイム、モデル、初回ダウンロードまたは同梱物の扱いが配布条件に影響する。PaddleOCR が未導入の Python を使うと OCR エラーになる。前処理や検出閾値は動画ごとに結果と処理時間へ影響する。 | 実 OCR 確認では PaddleOCR 導入済み Python を `MOVIE_TELOP_PADDLEOCR_PYTHON` で指定する。サンプル sidecar 検証では `MOVIE_TELOP_OCR_ENGINE=json-sidecar` を明示し、`materialize_sidecars.py` で sidecar を生成してから `OCR 再実行` を実施する。PaddleOCR の拡大率は `1.0` 固定とし、前処理、検出閾値、文字方向分類は設定画面から調整する。 | `#102`、`#72`、`#48`、`#49` | #48 で、アプリ本体 zip には `tools/ocr/paddle_ocr_worker.py` だけを同梱し、Python runtime / Python package / モデルは別手順または別配布物として扱う方針にした。#49 で導入手順へ反映する。 |
| C-002 | `materialize_sidecars.py` は `sample_basic_telop.mp4` 専用であり、任意動画の `frames/` に実行するとサンプル動画用 OCR 結果が書き込まれる。 | 任意動画の検証結果を誤って解釈する可能性がある。 | `test-data/basic_telop/README.md` の注意書きに従い、サンプル動画以外には使わない。 | 追加 Issue なし | 文書化済みの検証補助制約として扱う。 |
| C-003 | 初期リリースの `text_color`、`stroke_color`、`text_type` は、OCR bounding box 内の色分布から作る `白文字`、`黄文字`、`黒枠` などの暫定ラベルであり、厳密な `#RRGGBB` 色値や「人物名」「見出し」などの意味分類ではない。`font_family` と `background_color` は現行実装では `null` を許容する。 | 出力スキーマには属性項目が存在するが、実フォント、正確な代表色、背景帯、意味分類、ASS スタイル反映として扱うと解釈を誤る可能性がある。 | 初期リリースでは `docs/spec/07_テロップ属性リリース範囲.md` の範囲で補助情報として扱う。正確な色値、背景帯、意味分類、ASS スタイル反映は後続実装候補として扱う。 | 追加 Issue なし | #75 で初期リリース範囲として確定済み。 |
| C-004 | `font_size` は表示フォントサイズではなく OCR 矩形の高さ相当値として推定している。 | 実フォントサイズとして扱うと解釈を誤る可能性がある。 | 導入手順、リリースノート、出力仕様では「OCR 矩形高さ相当」として説明する。 | 追加 Issue なし | #75 で初期リリース範囲として確定済み。 |

## 7. リリース工程への引き継ぎ
- #47 の時点では、テスト工程を止める重大不具合は残っていない。
- 実動画で sidecar なしに文字認識する機能は、Windows OCR baseline worker の結果を踏まえ、PaddleOCR PP-OCRv5 worker を採用して #72 の完了条件に扱う。
- 配布物の成立条件は、#48 で Release build 出力前提として確定済み。self-contained publish 出力は初期リリースでは使わず、再調査は初期リリース後の改善候補にする。
- 属性推定の範囲は #75 で初期リリース範囲と既知制約に切り分け済み。導入手順とリリースノートでは、`font_size` が OCR 矩形高さ相当であること、色/枠が暫定ラベルであること、ASS は標準スタイルのみであることを記載する。
