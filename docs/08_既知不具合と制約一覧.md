# 既知不具合と制約一覧

## 1. 文書情報
- 対象 Issue: `#47`
- 作成日: 2026-04-28
- 対象工程: テスト
- 記録者: Codex

## 2. 目的
テスト工程で確認した不具合、既知制約、回避策、後続 Issue を整理し、リリース工程での判断材料にする。

## 3. サマリ
- テスト工程の主要シナリオは通過しており、アプリ本体の起動、動画入力、フレーム抽出、OCR sidecar 読み込み、属性解析、出力、ログ、段階的再実行は成立している。
- 現時点で、テスト工程を継続不能にする重大不具合は残っていない。
- リリース工程では、実 OCR worker、配布構成、導入手順、属性推定の扱いを確定する必要がある。

## 6. 後続 Issue
- `#72` 実 OCR ワーカーで実動画の文字認識を成立させる。
- `#90` QCDS 評価レポートを継続運用する。
- `#98` リリース後に追加の実動画サンプルと改善評価を拡張する。

## 7. リリース工程への引き継ぎ
- #47 の時点では、テスト工程を止める重大不具合は残っていない。
- 実動画で sidecar なしに文字認識する機能は、Windows OCR baseline worker の結果を踏まえ、PaddleOCR PP-OCRv5 worker を採用して #72 の完了条件に扱う。
- 配布物の成立条件は、#48 で Release build 出力前提として確定済み。self-contained publish 出力は初期リリースでは使わず、再調査は初期リリース後の改善候補にする。
- 属性推定の範囲は #75 で初期リリース範囲と既知制約に切り分け済み。導入手順とリリースノートでは、`font_size` が OCR 矩形高さ相当であること、色/枠が暫定ラベルであること、ASS は標準スタイルのみであることを記載する。
