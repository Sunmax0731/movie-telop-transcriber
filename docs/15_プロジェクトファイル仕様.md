# プロジェクトファイル仕様

## 1. 目的
- 一度処理した動画について、毎回フレーム抽出や OCR をやり直さずにタイムラインとプレビューを再表示できるようにする。
- 編集済みタイムライン、抽出済みフレーム、OCR 結果、表示設定を 1 ファイルにまとめ、再確認や持ち運びをしやすくする。

## 2. 利用フロー
1. 動画を読み込み、フレーム抽出と OCR を完了する。
2. メニューバーの `ファイル > プロジェクトを保存` を選ぶ。
3. `.mtproj` ファイルとして保存する。
4. 後から再表示したいときは `ファイル > プロジェクトを開く` を選び、保存済み `.mtproj` を開く。
5. 読み込み後は、タイムライン、検出結果一覧、プレビュー用フレーム、選択中セグメントを復元する。

## 3. 保存形式
- 拡張子は `.mtproj` とする。
- 実体は ZIP 形式の単一ファイルとする。
- project 内には、再表示に必要な run 成果物を同梱する。

### 3.1 同梱するもの
- `project.json`
  - 保存日時
  - アプリケーションバージョン
  - 元動画の参照パス
  - 元動画が保存時に存在したかどうか
  - run ID
  - 選択中セグメント ID / detection ID
  - UI 設定のスナップショット
- `bundle/<run_id>/frames/`
  - 抽出済みフレーム PNG
- `bundle/<run_id>/ocr/`
  - OCR worker の response JSON
- `bundle/<run_id>/attributes/`
  - 表示属性解析結果 JSON
- `bundle/<run_id>/output/`
  - `segments.json`
  - `segments.csv`
  - `frames.csv`
  - `segments.srt`
  - `segments.vtt`
  - `segments.ass`

### 3.2 同梱しないもの
- 元動画ファイル本体
  - `project.json` に参照パスだけを保存する。
- 実行時ログ
  - 読み込み後の再表示に必須ではないため、現時点では project に含めない。

## 4. 読み込み時の挙動
- project 内の `segments.json`、OCR response、属性解析結果からタイムラインを再構成する。
- プレビューは同梱済みフレーム画像を使って再表示する。
- 元動画の参照パスが現在も存在する場合は、そのパスを UI に表示する。
- 元動画の参照パスが失われていても、project に同梱されたフレームとタイムラインは読み込む。
- 元動画の再指定ダイアログは現時点では出さない。状態表示で「元動画パスが見つからないが project は読み込めた」ことを利用者へ伝える。

## 5. UI 設定の扱い
- 次の設定を project 保存時のスナップショットとして保持する。
  - 表示言語
  - フレーム抽出間隔
  - 出力先ルート
  - メインウィンドウサイズ
- project を開いた直後は、この設定を現在セッションへ反映する。

## 6. 現時点の制約
- `プロジェクトを保存` は、少なくとも 1 回以上フレーム抽出と OCR を完了した後で使う。
- `プロジェクトを開く` は `.mtproj` または ZIP 形式の同等内容を前提とする。
- project 読み込み後のログ表示パスは、元 run の `logs/` ではなく project 内 `output/` を中心に扱う。
