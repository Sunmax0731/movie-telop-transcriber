## 2. 目的

## 3. 結論
現行の標準配布方式は、`dotnet build -c Release -p:Platform=x64` で生成する WinUI 3 アプリの Release build 出力を含む zip と、`Install-MovieTelopTranscriber.ps1` / `Install-MovieTelopTranscriber.cmd` の組み合わせとする。

`dotnet publish --self-contained true` の出力は、#46 の旧検証に加えて 2026-04-30 の再検証でも 30 秒以内に `0xC000027B` で終了したため、現時点でも配布候補にしない。詳細は [test-results/2026-04-30_配布方式と導入構成の再評価.md](test-results/2026-04-30_配布方式と導入構成の再評価.md) を参照する。

PaddleOCR は実動画 OCR の既定エンジンとして採用する。ただし、初期リリースのアプリ本体 zip には Python ランタイム、PaddlePaddle / PaddleOCR の Python package、PaddleOCR モデル本体を同梱しない。アプリ本体には `tools/ocr/paddle_ocr_worker.py` を同梱し、利用者環境または別配布の OCR ランタイム一式を `MOVIE_TELOP_PADDLEOCR_PYTHON` で指定して接続する。

#114 以降は、導入ハードルを下げるため `Install-MovieTelopTranscriber.ps1` を配布対象に含める。インストーラはアプリ本体 zip の取得または展開済み zip の利用、所定ディレクトリへの配置、PaddleOCR Python 仮想環境の作成、Python package の導入、PaddleOCR モデル取得、起動スクリプト作成を行う。

## 3.1 配布方式の比較

| 方式 | 利点 | 欠点 | 判断 |
| --- | --- | --- | --- |
| Release build zip + installer | 現行実装と整合し、導入実測と起動実測がある | .NET 10 Desktop Runtime 前提、zip サイズが大きい | 採用 |
| self-contained publish | .NET runtime 前提を減らせる可能性がある | 再検証でも `0xC000027B` で終了した | 不採用 |
| アプリ zip + OCR runtime 分離配布 | app 更新と OCR runtime 更新を切り分けやすい | 利用者が複数資材を扱う必要がある | フォールバック候補 |
| 完全オフライン一括パック | 閉域端末へ持ち込みやすい | 配布サイズと検証コストが大きい | 現時点では採用しない |

## 4. 配布物の単位
### 4.1 必須配布物
| 配布物 | 形式 | 位置付け |
| --- | --- | --- |
| アプリ本体 | zip | WinUI 3 GUI アプリ、Windows App SDK / OpenCVSharp など build 出力一式、PaddleOCR worker script、利用者向け docs を含む |
| インストーラ | ps1 | オンライン導入用。アプリ本体 zip、PaddleOCR package、OCR モデルを取得し、既定ディレクトリへ配置する |
| 導入手順書 | Markdown | #49 で作成し、アプリ本体 zip 内の `docs/` と GitHub Release 本文から参照する |
| リリースノート | Markdown | #50 で作成し、GitHub Release 本文へ反映する |

### 4.2 任意配布物
| 配布物 | 形式 | 位置付け |
| --- | --- | --- |
| PaddleOCR ランタイムパック | zip | Python 仮想環境、Python package、PaddleOCR モデルをオフライン導入用にまとめる場合の別配布物。標準配布ではなく、必要になった時点で別 asset として扱う |
| サンプルデータ | zip またはアプリ本体 zip 内 `samples/` | `sample_basic_telop.mp4` など、初回動作確認用の最小サンプル。権利とサイズを確認できるものだけを含める |

## 5. アプリ本体 zip のフォルダ構成
#51 で作成するアプリ本体 zip は、次の構成を基準にする。

```text
movie-telop-transcriber-win-x64-v<version>/
  Install-MovieTelopTranscriber.ps1
  app/
    MovieTelopTranscriber.App.exe
    MovieTelopTranscriber.App.dll
    MovieTelopTranscriber.App.deps.json
    MovieTelopTranscriber.App.runtimeconfig.json
    MovieTelopTranscriber.App.pri
    *.xbf
    Assets/
    tools/
      ocr/
        paddle_ocr_worker.py
    runtimes/
    win-x64/
    Microsoft*.dll
    Microsoft*.winmd
    Microsoft*.pri
    OpenCvSharp*.dll
    onnxruntime*.dll
    WinRT.Runtime.dll
    その他 Release build 出力で生成された実行時依存ファイル
  docs/
    README.md
    08_既知不具合と制約一覧.md
    09_Windows_OCRワーカー導入手順.md
    10_PaddleOCRワーカー導入手順.md
    11_配布構成と同梱物.md
    12_導入手順書.md
    13_リリースノート.md
    spec/
      04_出力仕様.md
      06_QCDS評価仕様.md
      07_テロップ属性リリース範囲.md
    templates/
      qcds_evaluation_report_template.md
  samples/
```

`app/` 配下は Release build 出力ディレクトリの内容を基準にする。手動で DLL を選別すると WinUI 3、Windows App SDK、OpenCVSharp、ONNX Runtime の依存を欠落させるリスクがあるため、#51 の配布物作成では build 出力一式を staging へコピーしたうえで、除外対象だけを明示的に取り除く。

## 6. 同梱物
### 6.1 アプリ本体へ同梱するもの
| 区分 | 同梱対象 | 理由 |
| --- | --- | --- |
| アプリ実行ファイル | `MovieTelopTranscriber.App.exe`、`MovieTelopTranscriber.App.dll`、`*.deps.json`、`*.runtimeconfig.json`、`*.pri`、`*.xbf` | アプリ起動と画面表示に必要 |
| WinUI / Windows App SDK 関連 | `Microsoft*.dll`、`Microsoft*.winmd`、`Microsoft*.pri`、`Microsoft.UI.Xaml/`、多言語リソースフォルダ、`win-x64/` | 現行 build で起動確認した runtime 依存一式 |
| OpenCV / 画像処理関連 | `OpenCvSharp*.dll`、`runtimes/`、`onnxruntime*.dll`、`DirectML.dll` | 動画読み込み、フレーム抽出、画像処理に必要 |
| アプリ資産 | `Assets/`、`AppIcon.ico` 由来のロゴ類 | タイトルバー、ウィンドウ、アプリ表示に必要 |
| PaddleOCR worker script | `tools/ocr/paddle_ocr_worker.py` | アプリから PaddleOCR Python 環境へ接続するために必要 |
| インストーラ | `Install-MovieTelopTranscriber.ps1` | 利用者がアプリ配置、OCR runtime 準備、モデル取得をまとめて実行できるようにするため |
| 利用者向け文書 | `README.md`、導入手順、既知制約、出力仕様、属性範囲 | オフライン導入後も参照できるようにする |
| 最小サンプル | `sample_basic_telop.mp4`、`ground_truth.json`、サンプル README | 初回起動と出力確認のため |

### 6.2 アプリ本体 zip には同梱しないもの
| 区分 | 非同梱対象 | 理由 |
| --- | --- | --- |
| Python ランタイム | `python.exe`、Python 標準ライブラリ、仮想環境 | サイズが大きく、利用環境ごとの検証が必要なため、アプリ本体とは分離する。オンライン導入ではインストーラが利用者環境に作成する |
| PaddlePaddle / PaddleOCR package | `paddlepaddle`、`paddleocr`、依存 Python package | CPU / GPU、Python version、wheel 取得元の違いがあるため、別手順またはインストーラで扱う |
| PaddleOCR モデル | `PP-OCRv5_server_det`、`PP-OCRv5_server_rec` など | 初回ダウンロードまたは事前配置が必要。オンライン導入ではインストーラが worker の `--warmup-models` で取得する |
| 開発用ファイル | `src/`、`.git/`、`obj/`、`temp/`、`work/`、`docs/test-results/`、`*.pdb` | 利用者の実行には不要。ログや検証結果は別途 docs に要点だけ含める |
| 個別検証データ | `test-data/basic_telop/botirist.mp4` など未追跡または権利確認が未確定の動画 | 配布権利とサイズを確定できないものは含めない |
| サンプル sidecar 生成スクリプト | `materialize_sidecars.py` | 開発検証用の補助スクリプトであり、任意動画に使うとサンプル OCR 結果を書き込むため |

## 7. 初期ファイルと設定の扱い
- アプリの初期設定はコード内の既定値を使う。初期リリースでは standalone の設定ファイルを同梱しない。
- 出力先フォルダは利用者が画面から指定する。アプリは解析開始前に作成と書き込み確認を行い、失敗時は `OUTPUT_ROOT_UNAVAILABLE` として表示する。
- 実動画 OCR は既定で PaddleOCR を使う。導入手順では `MOVIE_TELOP_PADDLEOCR_PYTHON` と `MOVIE_TELOP_PADDLEOCR_DEVICE=cpu` を明示する。
- `MOVIE_TELOP_OCR_ENGINE=json-sidecar` はサンプル sidecar 検証時だけ指定する。sidecar がない実動画確認では使わない。
- インストーラ利用時は `Start-MovieTelopTranscriber.ps1` を作成し、PaddleOCR 用環境変数を設定してからアプリを起動する。
- 設定画面で変更できる PaddleOCR 前処理、検出しきい値、文字方向分類、文書ゆがみ補正は、利用者が動画ごとに調整する値として扱う。

## 8. 配布サイズ
2026-04-29 時点のローカル Release build 出力は次の規模だった。

| 対象 | ファイル数 | 展開サイズ |
| --- | ---: | ---: |
| `src/MovieTelopTranscriber.App/bin/x64/Release/net10.0-windows10.0.26100.0` | 1084 | 605.18 MB |
| `tools/ocr/paddle_ocr_worker.py` | 1 | 16,470 bytes |
| Windows OCR fallback worker の Release build 出力 | 7 | 25.83 MB |

主なサイズ要因は `win-x64/`、`runtimes/`、Windows App SDK / WinUI 関連 DLL、ONNX Runtime、DirectML、OpenCVSharp である。#51 の zip 作成時に、最終的な圧縮後サイズを release asset の検査結果として記録する。

#51 の最終的な zip サイズ、展開後サイズ、checksum は、配布物作成後の検証結果と GitHub Release の記録で扱う。

## 9. 配布制約と後続判断
| ID | 制約 | 初期リリースでの扱い | 後続判断 |
| --- | --- | --- | --- |
| D-001 | self-contained publish 出力は再検証でも起動失敗する | 継続して使わず、Release build 出力を使う | Windows App SDK 側の前提が変わった時点でのみ再評価する |
| D-002 | `.runtimeconfig.json` は .NET 10 runtime を参照する | #49 で .NET 10 Desktop Runtime の確認手順を記載する | runtime 同梱可否は別途検証 |
| D-003 | PaddleOCR の Python runtime とモデルをアプリ本体に含めない | #49 / #114 でオンライン導入とオフライン事前配置の両方を説明する。標準経路ではインストーラが取得する | 必要になった時点で OCR ランタイムパックを別 asset として追加する |
| D-004 | CPU 推論は動画や設定により処理時間が大きくなる | QCDS レポートとリリースノートに処理時間の前提を記載する | ONNX Runtime 化やモデル軽量化を継続改善候補にする |
| D-005 | 属性推定は初期リリース範囲に制約がある | `docs/spec/07_テロップ属性リリース範囲.md` とリリースノートへ明記する | 正確な色値、背景帯、意味分類は後続候補 |

## 10. #49 / #50 / #51 への引き継ぎ
- #49 では、この構成を前提に `docs/12_導入手順書.md` を作成し、`app/` の配置、PaddleOCR Python 環境の指定、オフラインモデル配置、初回起動確認、サンプル実行を手順化済み。
- #50 では、`docs/13_リリースノート.md` を作成し、self-contained publish 不採用、PaddleOCR runtime 非同梱、属性推定範囲、処理時間の前提を利用者向けの既知制約として整理済み。
- #51 では、`movie-telop-transcriber-win-x64-v<version>.zip` を作成し、zip 展開後の起動確認、`tools/ocr/paddle_ocr_worker.py` の存在確認、docs / samples の同梱確認、圧縮後サイズの記録を行った。
- #114 では、`Install-MovieTelopTranscriber.ps1` を zip 内と release asset 候補に追加し、モデル取得を worker の `--warmup-models` で明示実行できるようにする。
