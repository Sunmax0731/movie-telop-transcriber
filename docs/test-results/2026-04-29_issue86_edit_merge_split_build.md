## 実装対象
- タイムライン選択中の操作に `結合` と `分割` を追加したが、プレビュー同期の再検討が必要なため UI では一時的に非表示とする。
- 結合は選択行と次の編集可能行を対象にする。
- 分割は選択行のテキストを改行、空白、文字要素の順で自動的に二分する。
- 編集、削除後の現在結果を `_latestSegments` に反映し、detection 行の直接操作時は `_latestFrameAnalyses` も更新する。結合 / 分割は UI から非表示とする。
- `segments.json` に `edits` 配列を追加し、GUI 編集操作の履歴を追跡できるようにする。

## 手動確認観点
- タイムライン行選択時に、コピー、編集、削除ボタンが同一幅で表示され、結合 / 分割ボタンが表示されないこと。
- 編集後に `出力のみ` を実行すると、JSON / CSV に編集後テキストが出力されること。
- 削除後に `出力のみ` を実行すると、削除後の状態で JSON / CSV が出力されること。
- `segments.json` の `edits` に `edit`、`delete` の操作履歴が出力されること。`merge`、`split` は再設計後の確認対象とする。

## 追加修正
- セグメント行の結合 / 分割では OCR detection 原本を変更せず、統合済みセグメントだけを更新する。
- セグメント行のプレビュー detection はテキスト一致した場合に紐づけ、結合後の行では選択テキストに含まれる元 detection も強調対象にする。
- テキストが対応しない別 detection へはフォールバックしない。
- 手動確認で結合 / 分割後のプレビュー同期が安定しないことを確認したため、リリース前 UI では結合 / 分割ボタンを非表示にする。
