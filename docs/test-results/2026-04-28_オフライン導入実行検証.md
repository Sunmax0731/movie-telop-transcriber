# オフライン導入と実行検証結果

## 1. 文書情報
- 対象 Issue: `#46`
- 実施日: 2026-04-28
- 対象: Release self-contained publish の起動確認
- 記録者: Codex

## 2. 検証した手順と結果
### 2.1 self-contained publish
```powershell
dotnet publish src\MovieTelopTranscriber.App\MovieTelopTranscriber.App.csproj `
  -c Release `
  -p:Platform=x64 `
  -r win-x64 `
  --self-contained true `
  -o temp\offline-publish

Start-Process `
  -FilePath "D:\Claude\Movie\movie-telop-transcriber\temp\offline-publish\MovieTelopTranscriber.App.exe" `
  -WorkingDirectory "D:\Claude\Movie\movie-telop-transcriber\temp\offline-publish"
```

結果:
- 起動直後に終了した。

確認結果:
- `MovieTelopTranscriber.App.exe` の終了コードは `0xC000027B`。
- Windows の Application Error では `Microsoft.UI.Xaml.dll` の APPCRASH として記録された。
- Windows Error Reporting では `combase.dll`、`80004005` が記録された。

補足:
- `PublishTrimmed=false` を明示して clean publish しても、同じ `0xC000027B` で終了した。
- 一時的に `System.TypeLoadException: Could not load type 'System.AppContext'` が記録された publish 出力もあった。
- 現時点では `dotnet publish --self-contained true` の出力は #46 の成功条件に使わない。

### 2.2 Release build 出力
```powershell
dotnet build src\MovieTelopTranscriber.App\MovieTelopTranscriber.App.csproj `
  -c Release `
  -p:Platform=x64

Start-Process `
  -FilePath "D:\Claude\Movie\movie-telop-transcriber\src\MovieTelopTranscriber.App\bin\x64\Release\net10.0-windows10.0.26100.0\MovieTelopTranscriber.App.exe" `
  -WorkingDirectory "D:\Claude\Movie\movie-telop-transcriber\src\MovieTelopTranscriber.App\bin\x64\Release\net10.0-windows10.0.26100.0"
```

結果:
- 起動継続を確認した。
- #46 の現時点の手動確認は、この Release build 出力を使って継続する。

## 3. 判断
- self-contained publish 出力は起動失敗するため、現時点のオフライン導入候補としては未成立。
- Release build 出力は起動できるため、ローカル実行確認と手順洗い出しには利用できる。
- Release build 出力は .NET / Windows App SDK などの実行前提を利用環境側に要求する可能性があるため、同梱物とランタイム要件は #46 / #47 / リリース工程で整理する。
