## 2. 検証した手順と結果
### 2.1 self-contained publish
```powershell
dotnet publish src\MovieTelopTranscriber.App\MovieTelopTranscriber.App.csproj `
  -c Release `
  -p:Platform=x64 `
  -r win-x64 `
  --self-contained true `
  -o temp\offline-publish

Start-Process `
  -FilePath "D:\Claude\Movie\movie-telop-transcriber\temp\offline-publish\MovieTelopTranscriber.App.exe" `
  -WorkingDirectory "D:\Claude\Movie\movie-telop-transcriber\temp\offline-publish"
```

結果:
- 起動直後に終了した。

確認結果:
- `MovieTelopTranscriber.App.exe` の終了コードは `0xC000027B`。
- Windows の Application Error では `Microsoft.UI.Xaml.dll` の APPCRASH として記録された。
- Windows Error Reporting では `combase.dll`、`80004005` が記録された。

補足:
- `PublishTrimmed=false` を明示して clean publish しても、同じ `0xC000027B` で終了した。
- 一時的に `System.TypeLoadException: Could not load type 'System.AppContext'` が記録された publish 出力もあった。
- 現時点では `dotnet publish --self-contained true` の出力は #46 の成功条件に使わない。

### 2.3 Release build 出力での抽出実行
ユーザー手動確認で、`抽出` 実行後に進捗インジケータが途中停止し、処理状況に以下で始まるエラーが表示された。

```text
Reflection-based serialization has been disabled
```

確認結果:
- `work/runs/20260428_153626_7e5a/ocr/ocr-000001-00000000ms.request.json` が 0 byte で生成されていた。
- フレーム抽出後、最初の OCR request JSON を書き出す時点で `System.Text.Json` の reflection fallback が使われ、Release 実行時に停止していた。

対処:
- OCR request / response、属性解析結果、出力 package、run summary の JSON 入出力を `JsonSerializerOptions` ではなく source generation の `JsonTypeInfo<T>` に切り替えた。
- Release ビルドで `System.Text.Json` の reflection-based serialization 警告が出ないことを確認した。

### 2.6 サンプル sidecar 生成後の OCR 再実行
直近 run `20260428_155035_e28c` の `frames` に、`test-data/basic_telop/materialize_sidecars.py` でサンプル用 `.ocr.json` を生成したうえで、ユーザー手動確認により `OCR 再実行` を実施した。

結果:
- `work/runs/20260428_155035_e28c/logs/run.log` の `status` は `success` だった。
- `frame_count=5`、`detection_count=4`、`segment_count=2`、`warning_count=0`、`error_count=0` を確認した。
- `segments.json` に `サンプルテロップ` と `重要なお知らせ` の 2 セグメントが出力された。

補足:
- 初回抽出で `No text detected` になることは、外部 OCR worker 未設定かつ sidecar 未生成時の現行仕様と整合する。
- sidecar 生成後の `OCR 再実行` では、サンプルデータの期待テキストが出力されることを確認した。
