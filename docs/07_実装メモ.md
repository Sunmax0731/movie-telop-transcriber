# 実装メモ

## 1. 対象
実装工程で確定した実装状態と、後続工程で再評価する未確定事項を記録する。

## 2. OCR ワーカー接続
- アプリ本体は `IOcrWorkerClient` 経由で OCR を呼び出す。
- 初期実装は `ProcessOcrWorkerClient` を使用する。
- 外部ワーカーは `MOVIE_TELOP_OCR_WORKER` 環境変数で指定する。
- 外部ワーカーには `request.json` と `response.json` のパスを引数として渡す。
- 外部ワーカー未設定時は、抽出フレームと同名の `.ocr.json` を同じ契約のレスポンスとして読み込む。
- `.ocr.json` がない場合は空検出の `success` レスポンスを保存する。

## 3. 属性解析
- `TelopAttributeAnalysisService` が OCR レスポンスを内部属性モデルへ変換する。
- OCR 矩形から推定できる場合は `font_size` をピクセル高さ相当として設定する。
- `font_family`、`text_color`、`stroke_color`、`background_color` は初期実装では未確定値として `null` を許容する。
- `text_type` は初期値として `unknown` を設定する。

## 4. 中間成果物
- フレーム画像: `work/runs/<run_id>/frames/`
- OCR リクエストとレスポンス: `work/runs/<run_id>/ocr/`
- 属性解析結果: `work/runs/<run_id>/attributes/`

## 5. 後続工程で再評価する項目
- PaddleOCR / Tesseract / Windows 標準 OCR の採用実体。
- OCR モデルや外部ランタイムを配布物へ同梱する範囲。
- 色属性推定の画像解析方式。
- フレーム単位結果をセグメント単位へ統合する代表値算出方式。
