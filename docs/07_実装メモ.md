# 実装メモ

## 1. 対象
実装工程で確定した実装状態と、後続工程で再評価する未確定事項を記録する。

## 2. OCR ワーカー接続
- アプリ本体は `IOcrWorkerClient` 経由で OCR を呼び出す。
- `MOVIE_TELOP_OCR_ENGINE=paddleocr` の場合は `PaddleOcrWorkerClient` を使用する。
- それ以外の場合は `ProcessOcrWorkerClient` を使用する。
- 外部ワーカーは `MOVIE_TELOP_OCR_WORKER` 環境変数で指定する。
- 外部ワーカーには `request.json` と `response.json` のパスを引数として渡す。
- 外部ワーカー未設定時は、抽出フレームと同名の `.ocr.json` を同じ契約のレスポンスとして読み込む。
- `.ocr.json` がない場合は空検出の `success` レスポンスを保存する。
- PaddleOCR worker は `tools/ocr/paddle_ocr_worker.py` として追加済み。
- `PaddleOcrWorkerClient` は Python worker を stdio で常駐させ、PaddleOCR モデルを 1 回だけ読み込む。
- PaddleOCR worker は `rec_texts`、`rec_scores`、`rec_polys` を共通 OCR 契約の `OcrDetectionRecord` へ変換する。
- PaddleOCR worker は日本語向けに、`ちよ`、`つて`、`じや` などの小書き仮名誤認識を保守的に補正する。
- PaddleOCR worker は手動クロップを使わず、フルフレームに対する自動前処理としてコントラスト補正、シャープ化を適用できる。拡大率は `1.0` 固定とする。
- PaddleOCR の検出閾値、検出ボックス閾値、unclip ratio、limit side length、文字方向分類、文書ゆがみ補正は環境変数と設定画面から調整できる。
- `PaddleOcrWorkerClient` は PaddleOCR 関連の設定差分を検出した場合、常駐 worker を再起動して次の OCR 実行へ反映する。
- Windows 標準 OCR を使う baseline worker は `MovieTelopTranscriber.Ocr.Windows` として追加済み。
- Windows OCR worker は `Windows.Media.Ocr` を使い、行単位の検出結果を共通 OCR 契約へ変換する。

## 3. 属性解析
- `TelopAttributeAnalysisService` が OCR レスポンスを内部属性モデルへ変換する。
- OCR 矩形から推定できる場合は `font_size` をピクセル高さ相当として設定する。
- `font_family`、`text_color`、`stroke_color`、`background_color` は初期実装では未確定値として `null` を許容する。
- `text_type` は初期値として `unknown` を設定する。

## 4. 中間成果物
- フレーム画像: `work/runs/<run_id>/frames/`
- OCR リクエストとレスポンス: `work/runs/<run_id>/ocr/`
- 属性解析結果: `work/runs/<run_id>/attributes/`
- 実行ログ: `work/runs/<run_id>/logs/`

## 5. セグメント統合と出力
- `TelopSegmentMerger` が、近接フレーム内の類似文字列と互換属性を 1 セグメントへ統合する。
- 代表文字列はフレームごとの OCR confidence と出現回数を使って選択する。
- セグメントの終了時刻は、最後に検出されたフレーム時刻にフレーム抽出間隔を加算して算出する。
- `ExportPackageWriter` が `work/runs/<run_id>/output/` に以下を保存する。
- `segments.json`: 完全出力。
- `segments.csv`: セグメント単位 CSV。
- `frames.csv`: フレーム単位 CSV。
- OCR 検出が 0 件の場合も、空配列を含む JSON とヘッダー付き CSV を生成する。

## 6. 非モーダル子ウィンドウ
- `SettingsWindow` を詳細設定用の WinUI `Window` として実装する。
- メイン画面と同じ `MainPageViewModel` を参照する。
- 縦長化と重複を避けるため、設定画面は横長の 2 ペイン構成とする。
- 左ペインは言語設定と出力先フォルダ設定を扱う。
- 右ペインは PaddleOCR の前処理、検出閾値、文字方向分類、文書ゆがみ補正を扱う。
- 抽出間隔と OCR エンジンの読み取り表示はメイン画面に集約し、設定画面には置かない。
- PaddleOCR の前処理、検出閾値、文字方向分類、文書ゆがみ補正を同じ画面で調整する。
- 出力先フォルダは設定画面で編集し、ラン単位の出力、ログ、作業ディレクトリはメイン画面左ペインのステータスカードで確認する。
- 再オープン時は、閉じられていない既存ウィンドウを再利用して前面化する。

## 7. ラン単位ログ
- `RunLogWriter` が `logs/run.log` と `logs/summary.json` を生成する。
- `run.log` は人が確認しやすい key-value 形式とする。
- `summary.json` は機械処理しやすい JSON 形式とする。
- ログには入力動画、設定、件数、警告/エラー数、成果物パスを記録する。

## 8. エラーハンドリングと段階的再実行
- `LastFailedStageText`、`LastErrorCodeText`、`LastErrorMessageText` で直近失敗を GUI に表示する。
- 入力検証失敗、前段成果物不足、処理例外を区別して表示する。
- `Rerun frames` はフレーム抽出から後段まで再実行する。
- `Rerun OCR` は直近フレームを使って OCR 以降を再実行する。
- `Export only` は直近解析結果を使って出力とログ生成だけを再実行する。
- 直近ログファイルのパスを Last failure 表示と左ペインのステータスカードで確認できるようにする。

## 9. 後続工程で再評価する項目
- OCR モデルや PaddleOCR Python ランタイムを配布物へ同梱する範囲。

- 色属性推定の画像解析方式。
- フレーム単位結果をセグメント単位へ統合する代表値算出方式。

## 11. #87 / #74 / #88 UI/UX 改善
- `TimelineSegment` に detection ID、frame index、timestamp、confidence を持たせ、タイムライン上で OCR detection とプレビュー矩形を追跡できるようにした。
- 同一時刻の行は OCR bounding box の上端座標で並べ、フレーム内で上にある文字列を先に表示する。
- タイムラインは右ペインに配置し、時刻、フレーム、テキスト、詳細、認識精度の列構成とする。
- タイムラインの列見出しには境界線を表示し、区切り部分のドラッグで列幅を変更できる。
- 認識精度はパーセント表記と横方向ゲージで表示し、低い値を赤寄り、高い値を緑寄りで表現する。
- 中央ペイン下部へ処理状況、定量進捗、段階的再実行、直近失敗を集約した。
- 中央ペインと右ペインの間にドラッグ操作用の境界を設け、右ペイン幅を変更できる。
- 左ペインの出力、ログ、作業ディレクトリにはコピー操作を追加した。
- 設定画面は言語設定、出力先フォルダ設定、PaddleOCR 調整へ整理し、メイン画面と重複する現在設定サマリ、最新結果一覧、抽出間隔、OCR エンジン表示、個別出力パス操作を削除した。

# #85 実フレームプレビューと OCR 矩形
- `MainPage.xaml` のプレビュー領域を、空状態表示だけでなく `Image` と overlay `Canvas` を重ねる構成へ変更した。
- `MainPage.xaml.cs` で `PreviewImagePath` と `PreviewDetections` の変更を監視し、画像表示サイズに合わせて OCR polygon とラベルを再描画する。
- `MainPageViewModel` はタイムライン選択に合わせ、選択対象に最も近いフレーム解析結果をプレビューへ反映する。
- `PaddleOCR` worker の bounding box は元フレーム座標へ戻した値として扱う。UI 側では `Image.Stretch=Uniform` による表示スケールと余白だけを計算する。
- segment に対応する detection は、segment ID と認識文字列の照合で強調表示する。
- `icon.png` を元画像として `Assets/AppIcon.ico` とタイル用 PNG を生成し、`ApplicationIcon` と `AppWindow.SetIcon` の参照先を一致させる。

# #93 タイムライン編集とメイン画面設定整理
- `TimelineSegment` を `ObservableObject` 化し、テキスト列のインライン編集状態と認識精度ゲージ表示を行単位で管理する。
- 認識精度ゲージは 50% 未満を非表示、50% から 100% を 0 から 100 の幅へ正規化する。
- タイムライン選択に連動して編集 / 削除ボタンを表示し、編集時は対象行の `TextBox` へフォーカスする。
- 編集確定時は `_latestSegments`、`_latestFrameAnalyses`、`ResultRows`、`TimelineSegments` を更新し、削除時は該当 segment / detection を現在結果から除外する。
- 削除操作は WinUI `ContentDialog` で確認し、キャンセル時は現在結果を変更しない。
- プレビュー再生は `DispatcherTimer` で抽出済みフレームを順送りし、該当フレームのタイムライン行があれば選択状態も更新する。
- 左ペインは入力、設定ボタン、状態カードに整理し、状態カードのパス表示は出力フォルダのみとした。
- 設定画面は `OverlappedPresenter` でリサイズと最大化を不可にし、言語、抽出間隔、出力先フォルダ、検出設定へ責務を集約した。
- 出力先フォルダは解析開始前に絶対パス化、ディレクトリ作成、テスト書き込みを行い、失敗時は `OUTPUT_ROOT_UNAVAILABLE` としてフレーム抽出前に停止する。
