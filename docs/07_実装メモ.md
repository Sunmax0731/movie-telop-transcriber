# 実装メモ

## 1. 対象
実装工程で確定した実装状態と、後続工程で再評価する未確定事項を記録する。

## 2. OCR ワーカー接続
- アプリ本体は `IOcrWorkerClient` 経由で OCR を呼び出す。
- `MOVIE_TELOP_OCR_ENGINE=paddleocr` の場合は `PaddleOcrWorkerClient` を使用する。
- `MOVIE_TELOP_OCR_ENGINE` が未指定で `MOVIE_TELOP_OCR_WORKER` も未指定の場合は、実動画 OCR の既定として `PaddleOcrWorkerClient` を使用する。
- `MOVIE_TELOP_OCR_ENGINE=json-sidecar` など PaddleOCR 以外を明示した場合、または `MOVIE_TELOP_OCR_WORKER` を指定した場合は `ProcessOcrWorkerClient` を使用する。
- 外部ワーカーは `MOVIE_TELOP_OCR_WORKER` 環境変数で指定する。
- 外部ワーカーには `request.json` と `response.json` のパスを引数として渡す。
- `json-sidecar` 検証時は、抽出フレームと同名の `.ocr.json` を同じ契約のレスポンスとして読み込む。
- `json-sidecar` 検証時に `.ocr.json` がない場合は `OCR_SIDECAR_NOT_FOUND` の `error` レスポンスを保存し、空検出の `success` にはしない。
- PaddleOCR worker は `tools/ocr/paddle_ocr_worker.py` として追加済み。
- `PaddleOcrWorkerClient` は Python worker を stdio で常駐させ、PaddleOCR モデルを 1 回だけ読み込む。
- PaddleOCR worker は `rec_texts`、`rec_scores`、`rec_polys` を共通 OCR 契約の `OcrDetectionRecord` へ変換する。
- PaddleOCR worker は日本語向けに、`ちよ`、`つて`、`じや` などの小書き仮名誤認識を保守的に補正する。
- PaddleOCR worker は手動クロップを使わず、フルフレームに対する自動前処理としてコントラスト補正、シャープ化を適用できる。拡大率は `1.0` 固定とする。
- PaddleOCR の検出閾値、検出ボックス閾値、unclip ratio、limit side length、文字方向分類、文書ゆがみ補正は環境変数と設定画面から調整できる。
- `PaddleOcrWorkerClient` は PaddleOCR 関連の設定差分を検出した場合、常駐 worker を再起動して次の OCR 実行へ反映する。
- Windows 標準 OCR を使う baseline worker は `MovieTelopTranscriber.Ocr.Windows` として追加済み。
- Windows OCR worker は `Windows.Media.Ocr` を使い、行単位の検出結果を共通 OCR 契約へ変換する。

## 3. 属性解析
- `TelopAttributeAnalysisService` が OCR レスポンスを内部属性モデルへ変換する。
- OCR 矩形から推定できる場合は `font_size` をピクセル高さ相当として設定する。
- `font_family`、`background_color` は初期リリースでは未確定値として `null` を許容する。
- `text_color`、`stroke_color` は OCR bounding box 内の色分布から `白文字`、`黄文字`、`黒枠` などの暫定ラベルを設定する。
- `text_type` は初期値として `未分類` を設定し、色分布から推定できる場合は `白文字`、`緑文字`、`黒枠` などのラベルを設定する。

## 4. 中間成果物
- フレーム画像: `work/runs/<run_id>/frames/`
- OCR リクエストとレスポンス: `work/runs/<run_id>/ocr/`
- 属性解析結果: `work/runs/<run_id>/attributes/`
- 実行ログ: `work/runs/<run_id>/logs/`

## 5. セグメント統合と出力
- `TelopSegmentMerger` が、近接フレーム内の類似文字列と互換属性を 1 セグメントへ統合する。
- 代表文字列はフレームごとの OCR confidence と出現回数を使って選択する。
- セグメントの終了時刻は、最後に検出されたフレーム時刻にフレーム抽出間隔を加算して算出する。
- `ExportPackageWriter` が `work/runs/<run_id>/output/` に以下を保存する。
- `segments.json`: 完全出力。
- `segments.csv`: セグメント単位 CSV。
- `frames.csv`: フレーム単位 CSV。
- `segments.srt`: SRT 字幕。
- `segments.vtt`: WebVTT 字幕。
- `segments.ass`: 標準スタイルの ASS 字幕。
- OCR 検出が 0 件の場合も、空配列を含む JSON、ヘッダー付き CSV、空またはヘッダーのみの字幕ファイルを生成する。

## 6. 非モーダル子ウィンドウ
- `SettingsWindow` を詳細設定用の WinUI `Window` として実装する。
- メイン画面と同じ `MainPageViewModel` を参照する。
- 縦長化と重複を避けるため、設定画面は横長の 2 ペイン構成とする。
- 左ペインは言語設定と出力先フォルダ設定を扱う。
- 右ペインは PaddleOCR の前処理、検出閾値、文字方向分類、文書ゆがみ補正を扱う。
- 抽出間隔と OCR エンジンの読み取り表示はメイン画面に集約し、設定画面には置かない。
- PaddleOCR の前処理、検出閾値、文字方向分類、文書ゆがみ補正を同じ画面で調整する。
- 出力先フォルダは設定画面で編集し、ラン単位の出力フォルダだけをメイン画面左ペインのステータスカードで確認する。
- 再オープン時は、閉じられていない既存ウィンドウを再利用して前面化する。

## 7. ラン単位ログ
- `RunLogWriter` が `logs/run.log` と `logs/summary.json` を生成する。
- `run.log` は人が確認しやすい key-value 形式とする。
- `summary.json` は機械処理しやすい JSON 形式とする。
- ログには入力動画、設定、件数、警告/エラー数、成果物パスを記録する。

## 8. エラーハンドリングと段階的再実行
- `LastFailedStageText`、`LastErrorCodeText`、`LastErrorMessageText` で直近失敗を GUI に表示する。
- 入力検証失敗、前段成果物不足、処理例外を区別して表示する。
- `Rerun frames` はフレーム抽出から後段まで再実行する。
- `Rerun OCR` は直近フレームを使って OCR 以降を再実行する。
- `Export only` は直近解析結果を使って出力とログ生成だけを再実行する。
- 直近ログファイルのパスを Last failure 表示と左ペインのステータスカードで確認できるようにする。

# #93 タイムライン編集とメイン画面設定整理
- `TimelineSegment` を `ObservableObject` 化し、テキスト列のインライン編集状態と認識精度ゲージ表示を行単位で管理する。
- 認識精度ゲージは 50% 未満を非表示、50% から 100% を 0 から 100 の幅へ正規化する。
- タイムライン選択に連動して編集 / 削除ボタンを表示し、編集時は対象行の `TextBox` へフォーカスする。
- 編集確定時は `_latestSegments`、`_latestFrameAnalyses`、`ResultRows`、`TimelineSegments` を更新し、削除時は該当 segment / detection を現在結果から除外する。
- 削除操作は WinUI `ContentDialog` で確認し、キャンセル時は現在結果を変更しない。
- プレビュー再生は `DispatcherTimer` で抽出済みフレームを順送りし、該当フレームのタイムライン行があれば選択状態も更新する。
- 左ペインは入力、設定ボタン、状態カードに整理し、状態カードのパス表示は出力フォルダのみとした。
- 設定画面は `OverlappedPresenter` でリサイズと最大化を不可にし、言語、抽出間隔、出力先フォルダ、検出設定へ責務を集約した。
- 出力先フォルダは解析開始前に絶対パス化、ディレクトリ作成、テスト書き込みを行い、失敗時は `OUTPUT_ROOT_UNAVAILABLE` としてフレーム抽出前に停止する。

# #86 OCR 結果の編集・結合・分割 UI
- タイムライン操作ボタンに結合と分割を追加したが、プレビュー同期の再検討が必要なため、リリース前 UI では一時的に非表示とする。
- 結合時はセグメント行の先頭 `segment_id` を維持し、次セグメントを削除して終了時刻、confidence、source frame count を更新する。detection 行では先頭 detection のテキストを更新し、次 detection を削除する。
- 分割時は改行、空白、文字要素の順でテキストを二分する。セグメント行では元 `segment_id` と派生 `segment_id` の 2 行へ分け、detection 行では同じ bounding box を持つ派生 detection を追加する。
- セグメント行の結合 / 分割では OCR detection 原本を変更せず、プレビューは元 OCR 結果からテキスト一致した detection を優先してフォーカス対象にする。結合後の行では、選択テキストに含まれる元 detection も強調対象にする。
- `MainPageViewModel` は GUI 編集操作を `_latestSegments` に反映し、detection 行の直接操作時のみ `_latestFrameAnalyses` も更新する。`出力のみ` で現在結果を書き出せるようにする。
- `ExportPackage` に `edits` を追加し、`EditOperationRecord` として `edit`、`delete`、`merge`、`split` の履歴を JSON 出力へ保存する。

# #89 SRT / VTT / ASS 字幕出力
- `ExportPackageWriter` は `segments` から `segments.srt`、`segments.vtt`、`segments.ass` を生成する。
- SRT / VTT / ASS は UTF-8 no BOM、CRLF 改行で出力する。
- SRT は `HH:MM:SS,mmm`、VTT は `HH:MM:SS.mmm`、ASS は `H:MM:SS.cc` の時刻形式を使う。
- ASS は初期標準スタイル `Default` のみとし、現行リリースではテロップ属性を ASS スタイルへ反映しない。
- `RunSummaryRecord` と `run.log` に字幕ファイルパスを追加し、設定画面の現在設定サマリに `JSON / CSV / SRT / VTT / ASS` の出力形式を表示する。

# #102 OCR エンジン未設定時の誤起動防止
- `OcrWorkerClientFactory` は OCR エンジン未指定かつ外部 worker 未指定の場合、`json-sidecar` ではなく `PaddleOcrWorkerClient` を選択する。
- `json-sidecar` は `MOVIE_TELOP_OCR_ENGINE=json-sidecar` など PaddleOCR 以外のエンジン名を明示した場合の検証モードとして扱う。
- `ProcessOcrWorkerClient` は外部 worker も sidecar も存在しない場合、`OCR_SIDECAR_NOT_FOUND` の OCR エラー応答を保存する。
- `MainPageViewModel` は OCR エラーを検出した場合、直近の失敗を `OCR` として表示し、処理状況にも最新エラーコードを表示する。

# #90 QCDS 評価レポート
- `tools/validation/evaluate_qcds_report.py` は、正解データと `segments.json` を比較して QCDS 評価レポートを生成する。
- セグメント対応付けは、正規化後テキストの類似度と開始/終了時刻の近さを組み合わせて行う。
- 計測値は、文字列完全一致率、文字単位一致率、欠落、余計な検出、開始/終了時刻誤差、処理時間、警告/エラー件数とする。
- `--metrics-output` で機械可読 JSON を保存し、次回以降は `--previous-metrics` で改善前後の差分を比較できるようにする。
- 初期代表レポートは `test-data/basic_telop/sample_basic_telop.mp4` の PaddleOCR 実行結果 `20260429_033541_3031` を対象に作成する。

# #75 テロップ属性リリース範囲
- 初期リリースで保証する属性範囲は `docs/spec/07_テロップ属性リリース範囲.md` に整理する。
- `font_size` は OCR bounding box の高さ相当値であり、実フォントサイズではない。
- `text_color`、`stroke_color`、`text_type` は分類ラベルとして扱い、厳密な色値や意味分類ではない。
- `font_family` と `background_color` は現行実装では `null` を許容する既知制約とする。
- SRT / VTT はテキストと時刻のみ、ASS は標準スタイルのみを初期リリース範囲とする。
