# 実装メモ

## 1. 対象
実装工程で確定した実装状態と、後続工程で再評価する未確定事項を記録する。

## 2. OCR ワーカー接続
- アプリ本体は `IOcrWorkerClient` 経由で OCR を呼び出す。
- `MOVIE_TELOP_OCR_ENGINE=paddleocr` の場合は `PaddleOcrWorkerClient` を使用する。
- それ以外の場合は `ProcessOcrWorkerClient` を使用する。
- 外部ワーカーは `MOVIE_TELOP_OCR_WORKER` 環境変数で指定する。
- 外部ワーカーには `request.json` と `response.json` のパスを引数として渡す。
- 外部ワーカー未設定時は、抽出フレームと同名の `.ocr.json` を同じ契約のレスポンスとして読み込む。
- `.ocr.json` がない場合は空検出の `success` レスポンスを保存する。
- PaddleOCR worker は `tools/ocr/paddle_ocr_worker.py` として追加済み。
- `PaddleOcrWorkerClient` は Python worker を stdio で常駐させ、PaddleOCR モデルを 1 回だけ読み込む。
- PaddleOCR worker は `rec_texts`、`rec_scores`、`rec_polys` を共通 OCR 契約の `OcrDetectionRecord` へ変換する。
- PaddleOCR worker は日本語向けに、`ちよ`、`つて`、`じや` などの小書き仮名誤認識を保守的に補正する。
- PaddleOCR worker は手動クロップを使わず、フルフレームに対する自動前処理として拡大、コントラスト補正、シャープ化を適用できる。
- PaddleOCR の検出閾値、検出ボックス閾値、unclip ratio、limit side length、文字方向分類、文書ゆがみ補正は環境変数と設定画面から調整できる。
- `PaddleOcrWorkerClient` は PaddleOCR 関連の設定差分を検出した場合、常駐 worker を再起動して次の OCR 実行へ反映する。
- Windows 標準 OCR を使う baseline worker は `MovieTelopTranscriber.Ocr.Windows` として追加済み。
- Windows OCR worker は `Windows.Media.Ocr` を使い、行単位の検出結果を共通 OCR 契約へ変換する。

## 3. 属性解析
- `TelopAttributeAnalysisService` が OCR レスポンスを内部属性モデルへ変換する。
- OCR 矩形から推定できる場合は `font_size` をピクセル高さ相当として設定する。
- `font_family`、`text_color`、`stroke_color`、`background_color` は初期実装では未確定値として `null` を許容する。
- `text_type` は初期値として `unknown` を設定する。

## 4. 中間成果物
- フレーム画像: `work/runs/<run_id>/frames/`
- OCR リクエストとレスポンス: `work/runs/<run_id>/ocr/`
- 属性解析結果: `work/runs/<run_id>/attributes/`
- 実行ログ: `work/runs/<run_id>/logs/`

## 5. セグメント統合と出力
- `TelopSegmentMerger` が、近接フレーム内の類似文字列と互換属性を 1 セグメントへ統合する。
- 代表文字列はフレームごとの OCR confidence と出現回数を使って選択する。
- セグメントの終了時刻は、最後に検出されたフレーム時刻にフレーム抽出間隔を加算して算出する。
- `ExportPackageWriter` が `work/runs/<run_id>/output/` に以下を保存する。
- `segments.json`: 完全出力。
- `segments.csv`: セグメント単位 CSV。
- `frames.csv`: フレーム単位 CSV。
- OCR 検出が 0 件の場合も、空配列を含む JSON とヘッダー付き CSV を生成する。

## 6. 非モーダル子ウィンドウ
- `SettingsWindow` を設定と出力を統合した WinUI `Window` として実装する。
- メイン画面と同じ `MainPageViewModel` を参照する。
- 抽出間隔と言語設定を双方向同期し、現在の設定サマリを表示する。
- PaddleOCR の前処理、検出閾値、文字方向分類、文書ゆがみ補正を同じ画面で調整する。
- 出力ディレクトリ、JSON / CSV パス、ログパス、最新結果一覧を同じ画面に表示する。
- 再オープン時は、閉じられていない既存ウィンドウを再利用して前面化する。

## 7. ラン単位ログ
- `RunLogWriter` が `logs/run.log` と `logs/summary.json` を生成する。
- `run.log` は人が確認しやすい key-value 形式とする。
- `summary.json` は機械処理しやすい JSON 形式とする。
- ログには入力動画、設定、件数、警告/エラー数、成果物パスを記録する。

## 8. エラーハンドリングと段階的再実行
- `LastFailedStageText`、`LastErrorCodeText`、`LastErrorMessageText` で直近失敗を GUI に表示する。
- 入力検証失敗、前段成果物不足、処理例外を区別して表示する。
- `Rerun frames` はフレーム抽出から後段まで再実行する。
- `Rerun OCR` は直近フレームを使って OCR 以降を再実行する。
- `Export only` は直近解析結果を使って出力とログ生成だけを再実行する。
- 直近ログファイルのパスを Last failure 表示と設定/出力画面で確認できるようにする。

## 9. 後続工程で再評価する項目
- OCR モデルや PaddleOCR Python ランタイムを配布物へ同梱する範囲。
- PaddleOCR の速度最適化、ONNX Runtime 化、または Tesseract fallback の要否。
- 色属性推定の画像解析方式。
- フレーム単位結果をセグメント単位へ統合する代表値算出方式。
