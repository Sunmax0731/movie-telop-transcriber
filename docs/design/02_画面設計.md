# 画面設計

## 1. メイン画面構成
メイン画面は 3 ペイン構成とし、以下の責務を持たせる。

表示密度:
- 画面全体の主役は中央ペインのプレビューと右ペインのタイムラインとする。
- 左ペインは 340px を基準にし、右ペインはタイムライン作業幅を確保するためドラッグで幅を変更できる。
- 見出し、項目名、値、補足文で文字階層を分け、状態表示の値に過大な見出しスタイルを使わない。
- 左ペインの Status はスクロール可能なコンパクト一覧とし、長いパスは項目名と値を改行して表示する。
- 項目説明は常時表示しない。項目領域または入力欄にツールチップとして付与し、画面上は項目名と値を優先する。
- トップ領域にはアプリ名と概要文を置かず、動画パス、参照、抽出の操作だけを配置する。設定ボタンは左ペインの入力セクションに置く。
- 中央ペイン下部の再実行ボタンは、各言語の文字列が見切れないよう横3分割ではなく縦並びの全幅ボタンとする。
- 設定ボタンから開く子ウィンドウは、詳細設定用の非モーダル画面とし、メイン画面と同じ文字階層を使う。

### 左ペイン
目的:
- 入力と操作の集約

主な要素:
- 動画ファイル選択ボタン
- 入力動画情報表示
- 出力先表示
- 設定画面表示ボタン
- 処理状態表示

### 中央ペイン
目的:
- 現在の動画状態と処理状況の確認

主な要素:
- 動画プレビュー領域
- プレビュー再生 / 一時停止
- 再生位置表示
- 現在フレーム情報表示
- 選択中セグメント要約表示
- 処理状況、定量進捗、段階的再実行、直近失敗表示

### 右ペイン
目的:
- タイムライン作業面

主な要素:
- 時刻、フレーム、テキスト、詳細、認識精度の列
- 列境界のドラッグ幅変更
- 選択中テロップの編集と削除
- 縦横スクロール
- 右ペイン幅のドラッグ変更

## 2. タイムライン UI 構成
タイムライン領域は少なくとも以下で構成する。
- 動画全長を示す時間軸
- セグメント帯表示
- 現在再生位置カーソル
- 選択セグメントハイライト

補足:
- 初期実装では、秒単位目盛りを優先する。
- ズーム機能は設計余地として残すが、初期実装必須とはしない。

## 3. 一覧表示方針
セグメント一覧は表形式を基本とする。

初期列候補:
- 開始時刻
- 終了時刻
- テキスト
- 種別
- 信頼度

期待動作:
- 行選択でタイムラインとプレビューを同期する。
- ソート可能な余地を持たせる。

## 4. サブ画面設計
サブ画面は非モーダルとし、以下を分離する。

### 設定と出力画面
- 言語設定
- 出力先フォルダ設定
- PaddleOCR 前処理と検出パラメータ設定

初期実装:
- `SettingsWindow` として独立 `Window` で開く。
- メイン画面と同じ `MainPageViewModel` を参照する。
- 横長の 2 ペイン構成とし、左に言語、抽出間隔、出力先フォルダ、右に検出設定を配置する。
- OCR エンジン読み取り表示、最新結果一覧、個別出力パス操作はメイン画面と重複するため設定画面には置かない。
- `1680 x 720` の横長レイアウトを固定サイズとして扱い、リサイズと最大化は不可にする。
- 画面上部のタイトルと説明文は表示せず、左ペインのセクション名を `基本設定` とする。
- 言語設定の初期値は OS の UI 言語に合わせる。初期対応は日本語、英語、中国語、韓国語とし、未対応言語の場合は英語を選択する。
- 言語変更時は、メイン画面と設定画面の主要な固定ラベル、ボタン、Status 項目名を選択言語に更新する。

### エラー詳細画面
- エラーコード
- エラーメッセージ
- 回復可能性

### 属性詳細画面
- 色属性
- フォント属性
- 種別属性

## 5. 主要UIコンポーネント方針
WinUI 3 前提で、以下の標準コンポーネント採用を基本とする。
- `NavigationView` は初期画面 1 枚構成のため必須としない。
- `Grid` / `SplitView`
  - 3 ペイン構成のレイアウトに使用
- `TextBox` / `NumberBox`
  - 抽出間隔や時刻入力
- `Button`
  - 実行操作
- `InfoBar`
  - 実行状態、警告、エラー表示
- `ListView` または表形式相当コンポーネント
  - セグメント一覧
- `ContentDialog` ではなく独立ウィンドウ相当の非モーダル画面
  - サブ画面用途

## 6. 画面状態管理
GUI 状態は少なくとも以下を持つ。
- 入力未選択
- 動画読込済み
- 実行中
- 実行完了
- エラー発生

状態遷移時の基本挙動:
- 実行中は開始ボタンを無効化する。
- 実行中でも一覧参照とタイムライン確認は可能にする。
- エラー発生時は `InfoBar` とエラー詳細画面の両方で追跡できるようにする。
- 初期実装ではメイン画面の Activity 領域に `Rerun frames`、`Rerun OCR`、`Export only` を配置し、直近の失敗段階、エラーコード、エラーメッセージ、ログパスを表示する。

設定画面は 1 インスタンスを再利用する。既に開いている状態でボタンを押した場合は新規作成せず、既存ウィンドウを前面化する。
## 7. #87 / #74 / #88 画面責務の整理
- メイン画面の右ペインはタイムラインを主作業面とし、結果ペインの重複表示は廃止する。
- タイムラインは時刻、フレーム、テキスト、詳細、認識精度の列を持ち、テキストは複数行表示を許可する。
- 各列の境界は見出し上に線を表示し、ドラッグで幅変更できる操作対象として扱う。
- 認識精度は数値だけでなく、赤から緑の横方向ゲージで判別できるようにする。
- 中央ペイン下部は処理状況、定量進捗、段階的再実行、直近失敗の確認に限定する。
- 中央ペインと右ペインの境界はドラッグで幅変更できる。
- 左ペインの出力フォルダはコピー操作を持つステータスカードとして扱う。ログと作業ディレクトリは常設表示しない。
- 設定画面はメイン画面と重複する現在設定サマリ、最新結果一覧、OCR エンジン表示、個別出力パス操作を持たない。

## 8. #93 タイムライン編集と設定整理
- 認識精度ゲージは 50% を最低値、100% を最大値として描画し、50% 未満はゲージを非表示にする。
- タイムライン見出しの各ラベルは中央揃えにし、列境界線とドラッグ対象を視覚的に対応させる。
- タイムライン行を選択して編集ボタンを押すと、該当行のテキスト列をインライン編集状態にする。
- タイムライン行を選択して削除ボタンを押すと、削除とキャンセルを選べる確認ダイアログを表示する。
- プレビューには再生 / 一時停止ボタンを追加し、抽出済みフレームを順に確認できるようにする。
- 左ペインに表示するパスは出力フォルダに限定し、ログと作業フォルダは画面上の常設表示から外す。
- 言語設定、抽出間隔、出力先フォルダは設定画面に集約し、メイン画面左ペインからは削除する。
- 設定画面右ペインの見出しは `検出設定` とし、PaddleOCR の調整値を検出設定の一部として扱う。
- 出力先フォルダは解析開始前に作成と書き込み確認を行い、利用できない場合は処理状況へ明示する。

## 9. #95 追加改善
- プレビュー下部にシーケンスバーを配置し、抽出済みフレーム内の現在位置を変更できるようにする。
- 処理状況には経過時間と推定残り時間を表示し、言語設定に応じたラベルで 1 秒間隔更新する。
- 処理状況ペインの下部は左右 2 ペインとし、左に再実行ボタン、右に直近の失敗を配置する。
- タイムライン詳細列は認識精度や出現フレーム数と重複する表示を持たない。
- タイムライン選択時はコピー、編集、削除ボタンを同一幅で表示する。
- タイムライン見出し右クリックで列表示 ON/OFF メニューを開き、最低 1 列は表示状態を維持する。
- フレーム列は `Frame` 接頭辞を持たず、数値のみを表示する。
- 詳細列には `白文字`、`緑文字`、`黒枠` などのテロップ種類ラベルを表示する。
- 設定画面の抽出間隔と検出設定は、範囲を持つ項目をスライダーで調整する。
- 設定画面の設定セクションに初期設定へ戻すボタンを配置する。
- 設定画面の固定サイズは `1680 x 720` とする。

# #85 プレビューと OCR 矩形オーバーレイ
- 中央ペインのプレビュー領域は、抽出済みフレーム画像を `Image` で表示し、その上に overlay 用 `Canvas` を重ねる。
- OCR detection の bounding box は、元フレーム座標を表示サイズへスケーリングして polygon として描画する。
- 選択中の segment または result row に対応する detection は、通常の矩形より強い色と太い線で強調する。
- タイムライン選択は、同じフレーム参照情報を介してプレビューと同期する。
- フレーム未抽出、画像なし、OCR 未実行、画像ロード失敗の場合は、空状態メッセージを表示する。
