# 画面設計

## 1. メイン画面構成
メイン画面は 3 ペイン構成とし、以下の責務を持たせる。

表示密度:
- 画面全体の主役は中央ペインのプレビューとタイムラインとし、左ペインと右ペインは補助情報として扱う。
- 左ペインと右ペインは同じ幅にし、現行実装では右ペインに合わせて 340px とする。
- 見出し、項目名、値、補足文で文字階層を分け、状態表示の値に過大な見出しスタイルを使わない。
- 左ペインの Status はスクロール可能なコンパクト一覧とし、長いパスやログ情報は項目名と値を改行して表示する。
- 項目説明は常時表示しない。項目領域または入力欄にツールチップとして付与し、画面上は項目名と値を優先する。
- トップ領域にはアプリ名と概要文を置かず、動画パス、実行、設定、出力の操作だけを配置する。
- 右ペイン下部の再実行ボタンは、各言語の文字列が見切れないよう横3分割ではなく縦並びの全幅ボタンとする。
- 設定画面と出力画面もメイン画面と同じ文字階層を使い、非モーダル子ウィンドウ間で表示密度を揃える。

### 左ペイン
目的:
- 入力と操作の集約

主な要素:
- 動画ファイル選択ボタン
- 入力動画情報表示
- 抽出間隔入力
- 開始時刻入力
- 終了時刻入力
- 出力先表示
- 解析開始ボタン
- 解析停止ボタン
- 結果保存ボタン
- 処理状態表示
- 言語設定

### 中央ペイン
目的:
- 現在の動画状態と時間軸状態の確認

主な要素:
- 動画プレビュー領域
- 再生位置表示
- タイムライン表示領域
- 現在フレーム情報表示
- 選択中セグメント要約表示

### 右ペイン
目的:
- 抽出結果の確認と選択

主な要素:
- セグメント一覧
- 検索または絞り込み入力
- 選択中セグメント詳細
- 属性表示

## 2. タイムライン UI 構成
タイムライン領域は少なくとも以下で構成する。
- 動画全長を示す時間軸
- セグメント帯表示
- 現在再生位置カーソル
- 選択セグメントハイライト

補足:
- 初期実装では、秒単位目盛りを優先する。
- ズーム機能は設計余地として残すが、初期実装必須とはしない。

## 3. 一覧表示方針
セグメント一覧は表形式を基本とする。

初期列候補:
- 開始時刻
- 終了時刻
- テキスト
- 種別
- 信頼度

期待動作:
- 行選択でタイムラインとプレビューを同期する。
- ソート可能な余地を持たせる。

## 4. サブ画面設計
サブ画面は非モーダルとし、以下を分離する。

### 設定詳細画面
- OCR エンジン設定
- 作業ディレクトリ設定
- ログ設定
- 言語設定

初期実装:
- `SettingsWindow` として独立 `Window` で開く。
- メイン画面と同じ `MainPageViewModel` を参照する。
- 抽出間隔と言語設定は双方向同期し、OCR エンジン、入力動画、作業ディレクトリ、設定サマリは参照表示する。
- 言語設定の初期値は OS の UI 言語に合わせる。初期対応は日本語、英語、中国語、韓国語とし、未対応言語の場合は英語を選択する。
- 言語変更時は、メイン画面、設定画面、出力画面の主要な固定ラベル、ボタン、設定サマリ、Status 項目名を選択言語に更新する。

### 出力結果詳細画面
- JSON 出力結果の詳細確認
- CSV 出力内容の確認

初期実装:
- `ExportWindow` として独立 `Window` で開く。
- `segments.json`、`segments.csv`、`frames.csv` のパスと最新結果一覧を表示する。
- 解析前に開いた場合もメイン画面をブロックせず、解析完了後は同じ ViewModel から出力パスが更新される。

### エラー詳細画面
- エラーコード
- エラーメッセージ
- 回復可能性

### 属性詳細画面
- 色属性
- フォント属性
- 種別属性

## 5. 主要UIコンポーネント方針
WinUI 3 前提で、以下の標準コンポーネント採用を基本とする。
- `NavigationView` は初期画面 1 枚構成のため必須としない。
- `Grid` / `SplitView`
  - 3 ペイン構成のレイアウトに使用
- `TextBox` / `NumberBox`
  - 抽出間隔や時刻入力
- `Button`
  - 実行操作
- `InfoBar`
  - 実行状態、警告、エラー表示
- `ListView` または表形式相当コンポーネント
  - セグメント一覧
- `ContentDialog` ではなく独立ウィンドウ相当の非モーダル画面
  - サブ画面用途

## 6. 画面状態管理
GUI 状態は少なくとも以下を持つ。
- 入力未選択
- 動画読込済み
- 実行中
- 実行完了
- エラー発生

状態遷移時の基本挙動:
- 実行中は開始ボタンを無効化する。
- 実行中でも一覧参照とタイムライン確認は可能にする。
- エラー発生時は `InfoBar` とエラー詳細画面の両方で追跡できるようにする。
- 初期実装ではメイン画面の Activity 領域に `Rerun frames`、`Rerun OCR`、`Export only` を配置し、直近の失敗段階、エラーコード、エラーメッセージ、ログパスを表示する。

設定画面と出力画面は、それぞれ 1 インスタンスを再利用する。既に開いている状態でボタンを押した場合は新規作成せず、既存ウィンドウを前面化する。
