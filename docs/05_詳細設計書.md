# #85 プレビュー詳細設計
- `TimelineSegment` と `ResultRow` は、表示文字列に加えて `FrameIndex`、`TimestampMs`、`SegmentId`、必要に応じて `DetectionId` を持つ。
- `MainPageViewModel` は `PreviewImagePath`、`PreviewImageWidth`、`PreviewImageHeight`、`PreviewDetections` を公開する。
- `PreviewDetections` は `PreviewDetectionOverlay` として、OCR detection ID、認識文字、信頼度、bounding box、強調状態を保持する。
- segment 選択時は、segment の開始・終了時刻内で認識文字が一致するフレームを優先し、なければ detection を持つ近傍フレームを使う。
- overlay の座標変換は UI 側で実施し、OCR worker 側で元フレーム座標へ逆変換済みの値を前提にする。
- 画像ロードは `BitmapImage` に任せ、overlay は画像表示領域の `SizeChanged`、画像ロード完了、検出コレクション変更時に再描画する。
# #87 / #74 / #88 UI 作業面改善

## タイムライン表示モデル
- `TimelineSegment` は表示用の時刻範囲、文字列、詳細に加えて、`FrameIndex`、`TimestampMs`、`SegmentId`、`DetectionId`、`Confidence` を持つ。
- セグメント単位の行は、開始時刻、対応する OCR detection の bounding box 上端座標の順に並べる。
- セグメントが生成されない場合でも、フレーム単位の OCR detection を上端座標順にタイムラインへ表示する。
- 認識精度は `Confidence` からパーセント表記と横方向ゲージを生成し、低い値を赤寄り、高い値を緑寄りで表現する。

## メイン画面の責務
- 右側の結果ペインは廃止し、確認対象は右ペインのタイムラインへ集約する。
- 処理状況は中央ペイン下部の Activity ペインへ集約し、フレーム抽出、OCR、属性解析、出力、ログの各段階で定量的な進捗文字列を表示する。
- 中央ペインと右ペインの境界はドラッグで幅変更でき、右ペインのタイムライン作業幅を調整できる。
- 出力フォルダは左ペインのステータスカードからコピーでき、表示ボタンで Explorer から開ける。ログと作業ディレクトリは常設表示しない。

## 設定画面の責務
- 設定画面は言語設定、出力先フォルダ設定、PaddleOCR 調整へ整理する。
- メイン画面と重複していた現在設定サマリ、直近結果一覧、抽出間隔、OCR エンジンの読み取り表示は設定画面から削除する。
- 出力パス、JSON、CSV、ログ、サマリの個別 Copy / Open 操作はメイン画面と重複するため設定画面には置かない。

# #93 タイムライン編集と設定整理

## タイムライン編集モデル
- `TimelineSegment` は `ObservableObject` として扱い、表示テキストと編集状態を UI から変更できる。
- 選択中の行に `SegmentId` または `DetectionId` がある場合だけ、編集ボタンと削除ボタンを表示する。
- 編集ボタン押下時は該当行のテキスト列を `TextBox` に切り替え、フォーカスを移す。
- 編集確定時は `_latestSegments`、`_latestFrameAnalyses`、`ResultRows`、`TimelineSegments` の現在結果を更新する。
- 削除ボタン押下時は `ContentDialog` で確認し、削除が選択された場合だけ該当 segment / detection を現在結果から除去する。
- 編集または削除後のファイル反映は `出力のみ` の再実行で行う。

## 認識精度ゲージ
- 認識精度の数値は従来どおりパーセント表示する。
- ゲージは 50% から 100% を表示範囲とし、50% が左端、100% が右端になるよう正規化する。
- 50% 未満または値なしの場合はゲージ自体を非表示にする。
- 色は 50% 付近を赤寄り、100% を緑寄りとして線形補間する。

## プレビュー再生
- プレビュー領域には再生 / 一時停止ボタンを配置する。
- 再生中は `DispatcherTimer` で抽出済みフレームを順に進め、該当フレームにタイムライン行がある場合は選択状態も更新する。
- 抽出済みフレームがない場合は再生を開始せず、状態メッセージへ理由を表示する。

## メイン画面と設定画面の責務
- メイン画面上部の設定ボタンは左ペインの入力セクションへ移動する。
- 左ペインの状態カードは動画、フレーム、OCR、セグメント、出力フォルダに限定し、ログと作業フォルダは画面から除外する。
- 言語設定、抽出間隔、出力先フォルダは設定画面で変更する。
- 設定画面は `1680 x 720` を固定値として扱い、リサイズと最大化を不可にする。
- 設定画面右ペインのタイトルは `検出設定` とし、PaddleOCR 専用名を前面に出しすぎない。

## 出力先フォルダの事前検証
- 解析開始時に出力先フォルダを絶対パスへ正規化し、存在しない場合は作成する。
- 指定先がファイルの場合、またはテスト書き込みに失敗した場合は解析を開始しない。
- 失敗時は `OUTPUT_ROOT_UNAVAILABLE` として処理状況へ表示し、フレーム抽出前に原因を確認できるようにする。

# #95 タイムライン操作と検出設定 UI の追加改善

## OCR worker の日本語パス対応
- `PaddleOcrWorkerClient` は worker プロセスの `StandardInputEncoding`、`StandardOutputEncoding`、`StandardErrorEncoding` を UTF-8 に固定する。
- 日本語を含む Desktop 配下の request / response パスを stdio JSON で渡しても、Python worker が `UnicodeDecodeError` で終了しないようにする。
- PaddleOCR worker 側では、単独の `○`、`〇`、`◯`、および正方形に近い `O` / `0` を `◎` として保守的に正規化する。
- PaddleOCR が文字列を返さない検出領域でも、二重丸に近い形状の場合は `◎` として補完する。

## 処理状況表示
- フレーム抽出と OCR の進捗文字列には、処理済みフレーム数、総フレーム数、進捗率、経過時間、推定残り時間を表示する。
- 推定残り時間は、現在までの経過時間を処理済みフレーム数で割り、残りフレーム数に掛けて算出する。
- 経過時間、残り時間、フレーム単位などのラベルは、現在の言語設定に合わせて表示する。
- 実行中は 1 秒間隔で進捗文字列を再計算し、フレーム完了イベントがない間も時間表示を更新する。
- 処理状況ペインは中央ペイン下部のコンパクト表示とし、再実行ボタン 3 つは横並びで配置する。
- 直近の失敗は状態セクション側へ移動し、処理状況ペインは現在処理と再実行操作に限定する。

## プレビュー位置操作
- プレビューにはシーケンスバーを追加し、抽出済みフレーム配列の位置をスライダーで変更できるようにする。
- スライダー操作時は、該当フレームに対応するタイムライン行があれば選択状態を同期する。

## 設定画面
- 抽出間隔、コントラスト、検出しきい値、検出ボックスしきい値、unclip ratio、limit side length はスライダーで調整する。
- 設定セクションには初期設定へ戻すボタンを追加し、言語、抽出間隔、PaddleOCR 前処理、検出設定を既定値へ戻す。
- 設定画面の固定サイズは `1680 x 720` とする。
- 設定画面上部のタイトルと説明文はメイン画面の設定ボタンと重複するため表示しない。
- 設定画面の左ペインのセクション名は `基本設定` とする。
- 設定画面の PaddleOCR 前処理、しきい値、方向分類、文書ゆがみ補正のラベルと説明は言語設定に合わせて切り替える。

## タイムライン詳細列
- 認識精度は専用列で扱うため、詳細列から confidence 表示を除外する。
- 詳細列の見出しは表示内容に合わせて `表示属性` とし、色属性などを一行で表示する。
- 詳細列は同じ内容の二重表示を避け、認識精度と出現フレーム数以外の補助情報に限定する。

## タイムライン操作
- 選択中テロップがある場合は、コピー、編集、削除ボタンを同一幅で表示する。
- コピーは選択中テロップのテキストをクリップボードへ保存する。
- フレーム列は `Frame` 接頭辞を表示せず、6 桁の数値のみを表示する。
- タイトル行の右クリックで列表示メニューを開き、時刻、フレーム、テキスト、詳細、認識精度の表示 ON/OFF を切り替えられる。
- 列表示は最低 1 列を残す。

## #86 / #98 OCR 結果の編集・結合・分割
- タイムライン選択中の編集可能行に対し、コピー、編集、削除の操作を提供する。
- 結合は選択行と次の編集可能行を対象とし、テキストを空白区切りで連結する。時刻表示らしい文字列、または表示属性が異なる行同士は誤操作を避けるため結合対象外にする。セグメント行では先頭行の `segment_id` を維持し、終了時刻、confidence、source frame count を更新する。
- 分割は選択行のテキストを改行または空白で二分し、セグメント行では元 `segment_id` の行と派生 `segment_id` の行に分ける。文字要素単位の任意分割は、分割位置指定 UI を追加するまで行わない。
- 時刻表示らしい文字列はテロップ本文として分割しない。
- OCR detection のみで表示されている行でも、結合時は次 detection を削除し、分割時は同じ bounding box を持つ派生 detection を作る。
- タイムライン / 結果行は単一の代表 detection ID に加えて、編集後セグメントに関連する detection ID 群を保持する。プレビューはこの detection ID 群を優先して強調し、結合後の複数 detection や分割後の関連 detection を同じ選択規則で扱う。
- 結合 / 分割後のタイムライン再構築では、操作時点の関連 detection ID 群を引き継ぎ、テキスト再推定だけに依存しない。
- GUI 編集操作は `_latestSegments` に反映し、detection 行を直接操作する場合のみ `_latestFrameAnalyses` も更新する。`出力のみ` 実行時に JSON / CSV へ反映する。
- `segments.json` には `edits` 配列を追加し、`edit`、`delete`、`merge`、`split` の操作種別、対象 ID、関連 ID、元テキスト、更新後テキスト、操作時刻を保存する。
- 結合 / 分割ボタンは、編集可能なタイムライン行を選択している場合に表示する。

## #89 SRT / VTT / ASS 字幕出力
- `ExportPackageWriter` は JSON / CSV に加えて `segments.srt`、`segments.vtt`、`segments.ass` を `output` ディレクトリへ出力する。
- 字幕ファイルは `SegmentRecord` の `StartTimestampMs`、`EndTimestampMs`、`Text` から生成する。
- SRT / VTT / ASS は UTF-8 no BOM、CRLF 改行で出力する。
- ASS は初期標準スタイルのみを持つ。文字色、縁取り、配置などの詳細スタイル反映は後続の属性推定高度化で扱う。
- `RunSummaryRecord` と `run.log` には字幕ファイルパスを記録する。
- 設定画面の現在設定サマリには、出力形式として `JSON / CSV / SRT / VTT / ASS` を表示する。

## #100 プレビュー/タイムライン初期幅と処理状況高さリサイズ
- メイン画面の中央ペインと右ペインは、初期表示で同じ `*` 幅になるように `RightPaneWidth` の初期値を star sizing とする。
- 中央ペインと右ペインの境界ドラッグ後は、右ペインの実表示幅を基準にピクセル幅へ切り替えて resize する。
- 中央ペインはプレビュー行、水平 splitter 行、処理状況行の 3 行構成とする。
- 処理状況行は `ActivityPaneHeight` で管理し、初期値は従来どおり 160px とする。
- プレビューと処理状況の境界を上下にドラッグした場合、処理状況行の高さを更新する。最小高さは 150px とし、再実行ボタンが読める高さを下回らないようにする。

## #102 OCR エンジン未設定時の誤起動防止
- `OcrWorkerClientFactory` は `MOVIE_TELOP_OCR_ENGINE=paddleocr` の場合、または OCR エンジン未指定かつ外部 worker 未指定の場合に `PaddleOcrWorkerClient` を選択する。
- `MOVIE_TELOP_OCR_WORKER` が指定されている場合、または `MOVIE_TELOP_OCR_ENGINE=json-sidecar` など PaddleOCR 以外が明示されている場合は `ProcessOcrWorkerClient` を選択する。
- `json-sidecar` 検証時にフレーム同名の `.ocr.json` がない場合、`ProcessOcrWorkerClient` は `OCR_SIDECAR_NOT_FOUND` の OCR エラー応答を保存する。
- `MainPageViewModel` は OCR エラーを検出した場合、処理状況に OCR エラー件数と最新エラーコードを表示し、左ペインの直近失敗へ stage `OCR`、エラーコード、エラーメッセージを表示する。

## テロップ種類ラベル
- 属性解析では OCR bounding box 内の色分布から `白文字`、`緑文字`、`黒枠` などの暫定ラベルを生成する。
- 判定できない場合は `未分類` と表示し、従来の `unknown` 表示は使用しない。
- 初期リリースでは、`text_type` は意味分類ではなく表示属性ラベルとして扱う。見出し、人物名、通常テロップなどの分類は後続実装で扱う。
- `font_size` は OCR bounding box の高さ相当値であり、実フォントサイズとして扱わない。
- `font_family` と `background_color` は現行実装では `null` を許容する。

## #90 QCDS 評価レポート
- QCDS 評価はアプリ本体の出力 `segments.json`、`logs/summary.json` と、代表動画ごとの正解データを入力にする。
- 評価スクリプトは `tools/validation/evaluate_qcds_report.py` とし、アプリ本体の実行経路とは分離する。
- レポートは Markdown と機械可読 JSON の 2 形式で保存し、Markdown は人手レビュー、JSON は改善前後比較に使う。
- 比較対象はセグメント単位とし、テキスト類似度と時刻近接度から正解セグメントと実セグメントを対応付ける。
- 余計な検出は、対応付かなかった実セグメントとして記録する。背景文字や時計表示の混入を QCDS の品質指標として追跡する。
- プレビュー overlay、テロップ種別ラベル、GUI 操作感は自動評価値とは分けて、手動確認欄へ記録する。
