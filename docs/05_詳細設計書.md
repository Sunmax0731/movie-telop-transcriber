# #85 プレビュー詳細設計
- `TimelineSegment` と `ResultRow` は、表示文字列に加えて `FrameIndex`、`TimestampMs`、`SegmentId`、必要に応じて `DetectionId` を持つ。
- `MainPageViewModel` は `PreviewImagePath`、`PreviewImageWidth`、`PreviewImageHeight`、`PreviewDetections` を公開する。
- `PreviewDetections` は `PreviewDetectionOverlay` として、OCR detection ID、認識文字、信頼度、bounding box、強調状態を保持する。
- segment 選択時は、segment の開始・終了時刻内で認識文字が一致するフレームを優先し、なければ detection を持つ近傍フレームを使う。
- overlay の座標変換は UI 側で実施し、OCR worker 側で元フレーム座標へ逆変換済みの値を前提にする。
- 画像ロードは `BitmapImage` に任せ、overlay は画像表示領域の `SizeChanged`、画像ロード完了、検出コレクション変更時に再描画する。
# #87 / #74 / #88 UI 作業面改善

## タイムライン表示モデル
- `TimelineSegment` は表示用の時刻範囲、文字列、詳細に加えて、`FrameIndex`、`TimestampMs`、`SegmentId`、`DetectionId`、`Confidence` を持つ。
- セグメント単位の行は、開始時刻、対応する OCR detection の bounding box 上端座標の順に並べる。
- セグメントが生成されない場合でも、フレーム単位の OCR detection を上端座標順にタイムラインへ表示する。
- 認識精度は `Confidence` からパーセント表記と横方向ゲージを生成し、低い値を赤寄り、高い値を緑寄りで表現する。

## メイン画面の責務
- 右側の結果ペインは廃止し、確認対象は右ペインのタイムラインへ集約する。
- 処理状況は中央ペイン下部の Activity ペインへ集約し、フレーム抽出、OCR、属性解析、出力、ログの各段階で定量的な進捗文字列を表示する。
- 中央ペインと右ペインの境界はドラッグで幅変更でき、右ペインのタイムライン作業幅を調整できる。
- 出力、ログ、作業ディレクトリは左ペインのステータスカードからコピーできる。

## 設定画面の責務
- 設定画面は言語設定、出力先フォルダ設定、PaddleOCR 調整へ整理する。
- メイン画面と重複していた現在設定サマリ、直近結果一覧、抽出間隔、OCR エンジンの読み取り表示は設定画面から削除する。
- 出力パス、JSON、CSV、ログ、サマリの個別 Copy / Open 操作はメイン画面と重複するため設定画面には置かない。

# #93 タイムライン編集と設定整理

## タイムライン編集モデル
- `TimelineSegment` は `ObservableObject` として扱い、表示テキストと編集状態を UI から変更できる。
- 選択中の行に `SegmentId` または `DetectionId` がある場合だけ、編集ボタンと削除ボタンを表示する。
- 編集ボタン押下時は該当行のテキスト列を `TextBox` に切り替え、フォーカスを移す。
- 編集確定時は `_latestSegments`、`_latestFrameAnalyses`、`ResultRows`、`TimelineSegments` の現在結果を更新する。
- 削除ボタン押下時は `ContentDialog` で確認し、削除が選択された場合だけ該当 segment / detection を現在結果から除去する。
- 編集または削除後のファイル反映は `出力のみ` の再実行で行う。

## 認識精度ゲージ
- 認識精度の数値は従来どおりパーセント表示する。
- ゲージは 50% から 100% を表示範囲とし、50% が左端、100% が右端になるよう正規化する。
- 50% 未満または値なしの場合はゲージ自体を非表示にする。
- 色は 50% 付近を赤寄り、100% を緑寄りとして線形補間する。

## プレビュー再生
- プレビュー領域には再生 / 一時停止ボタンを配置する。
- 再生中は `DispatcherTimer` で抽出済みフレームを順に進め、該当フレームにタイムライン行がある場合は選択状態も更新する。
- 抽出済みフレームがない場合は再生を開始せず、状態メッセージへ理由を表示する。

## メイン画面と設定画面の責務
- メイン画面上部の設定ボタンは左ペインの入力セクションへ移動する。
- 左ペインの状態カードは動画、フレーム、OCR、セグメント、出力フォルダに限定し、ログと作業フォルダは画面から除外する。
- 言語設定、抽出間隔、出力先フォルダは設定画面で変更する。
- 設定画面は現在の横長サイズを固定値として扱い、リサイズと最大化を不可にする。
- 設定画面右ペインのタイトルは `検出設定` とし、PaddleOCR 専用名を前面に出しすぎない。

## 出力先フォルダの事前検証
- 解析開始時に出力先フォルダを絶対パスへ正規化し、存在しない場合は作成する。
- 指定先がファイルの場合、またはテスト書き込みに失敗した場合は解析を開始しない。
- 失敗時は `OUTPUT_ROOT_UNAVAILABLE` として処理状況へ表示し、フレーム抽出前に原因を確認できるようにする。
