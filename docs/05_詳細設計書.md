# 詳細設計書

## 1. 文書情報
- システム名: movie-telop-transcriber
- 文書種別: 詳細設計書
- 作成日: 2026-04-28
- 対象工程: 設計

## 2. 対象
本書では、OCR ワーカー連携、処理パイプライン、層間 I/O 契約、失敗時の戻り値、再実行単位を定義する。

## 3. 処理パイプライン
処理の標準順序は以下とする。

1. 動画入力検証
2. 動画メタデータ取得
3. フレーム抽出
4. OCR ワーカーによる文字領域検出
5. OCR ワーカーによる文字認識
6. 属性解析
7. セグメント統合
8. JSON / CSV 出力
9. GUI 表示更新

## 4. OCR ワーカー連携方針
### 4.1 基本方針
- GUI 本体は OCR エンジンの詳細実装を直接呼び出さない。
- アプリケーション制御層が OCR ワーカーを呼び出し、共通契約へ正規化して GUI と後続処理へ渡す。
- OCR ワーカーは将来的な差し替えを前提に、入出力を共通形式にする。

### 4.2 呼び出し単位
- 初期実装ではフレーム単位呼び出しを基本とする。
- 将来、複数フレームのバッチ呼び出しへ拡張できる構成にする。

### 4.3 ワーカー責務
- 入力画像読込
- 文字領域検出
- 文字認識
- 検出結果の正規化
- 信頼度の付与
- エラー内容の返却

## 6. ステータスと戻り値
### 6.1 共通ステータス
- `success`
- `warning`
- `error`

### 6.2 エラー返却形式
```json
{
  "code": "OCR_PROCESS_FAILED",
  "message": "OCR worker failed to process frame.",
  "details": "engine timeout",
  "recoverable": true
}
```

### 6.3 代表的なエラーコード
- `VIDEO_NOT_FOUND`
- `VIDEO_DECODE_FAILED`
- `FRAME_EXTRACT_FAILED`
- `OCR_PROCESS_FAILED`
- `ATTRIBUTE_ANALYSIS_FAILED`
- `SEGMENT_MERGE_FAILED`
- `EXPORT_FAILED`

## 7. 再実行単位
初期仕様では以下の単位で再実行可能とする。

1. 動画読込から全実行
2. フレーム抽出から再実行
3. OCR ワーカーから再実行
4. 属性解析から再実行
5. セグメント統合から再実行
6. 出力のみ再実行

再実行条件:
- 前段成果物が存在すること
- 入力設定の整合が取れていること
- ログ上で対象実行単位が追跡可能であること

## 8. ログ設計
### 8.1 ログに残す内容
- 実行開始時刻
- 実行終了時刻
- 動画ファイル名
- フレーム抽出設定
- OCR エンジン名
- 処理件数
- 警告件数
- エラー件数

### 8.2 ログ粒度
- 実行単位ログ
- フレーム単位ログ
- ワーカー呼び出し単位ログ

## 9. GUI 連携ポイント
- 処理開始時に進捗状態を `待機中` から `実行中` へ変更する。
- フレーム抽出完了件数を進捗表示へ反映する。
- OCR 失敗時は、対象フレームとエラー理由を GUI へ通知する。
- セグメント確定後は、結果一覧とタイムライン表示モデルを同時更新する。
