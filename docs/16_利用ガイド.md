# 利用ガイド

## 1. 目的
この文書は、`movie-telop-transcriber` を初めて使う利用者向けに、主な機能、画面の見方、向いている使い方を整理したガイドである。

インストールや起動手順は [12_導入手順書.md](12_導入手順書.md) を参照し、この文書では「何ができるか」「どの場面でどう使うか」に重点を置く。

## 2. このアプリでできること
- 動画から一定間隔でフレームを抽出し、テロップ候補を OCR で読み取る。
- 認識した文字列を、動画時刻とフレーム番号に紐づけて確認する。
- テキスト、表示属性、推定サイズ、認識精度をタイムラインで見比べる。
- プレビュー上で対象テロップの位置を確認しながら、タイムラインと同期して見直す。
- 誤認識したテキストを編集し、SRT / VTT / ASS / CSV / JSON へ出力する。
- `.mtproj` プロジェクトファイルとして保存し、後日同じ状態から再確認する。

## 3. 向いているユースケース
### 3.1 字幕作成のたたき台を作る
動画からテロップを抽出し、`segments.srt` や `segments.vtt` を作って、あとで人手で整える使い方に向いている。

向いている場面:
- 配信アーカイブから字幕原稿の初期データを作る
- 社内動画の字幕化前に、表示テロップを一覧化する
- 同じ形式の動画をまとめて処理し、字幕作成の下準備をする

### 3.2 動画内テロップの確認表を作る
`segments.csv` や `frames.csv` を出力し、「どの時刻に何が出ていたか」を表形式で確認したい場合に向いている。

向いている場面:
- 動画編集後のテロップ抜け漏れ確認
- 放送や配信前の表記監査
- 時刻付きでレビュー依頼を出したいときの証跡作成

### 3.3 見た目属性も含めて点検する
表示属性、サイズ、認識精度を見ながら、どのテロップが不安定かを絞り込みたい場合に向いている。

向いている場面:
- 小さい文字や背景と重なる文字の検出漏れ確認
- 色付きテロップや縁取り付き文字の認識状態確認
- OCR 設定を調整しながら再実行して比較する作業

### 3.4 途中状態を保存して後日再開する
抽出済みフレームや OCR 結果を `.mtproj` へ保存し、次回は同じ解析結果を再表示したい場合に向いている。

向いている場面:
- 長尺動画を数日に分けて確認する
- 別担当者へ確認作業を引き継ぐ
- 一度出した結果を、設定変更後と比較する

## 4. 基本画面の見方
### 4.1 起動直後のメイン画面
![起動直後のメイン画面](images/app-main-empty.png)

主な役割:
- 上部: 入力動画の選択と抽出開始
- 左ペイン: 入力動画、フレーム数、OCR 件数、出力先、直近失敗の確認
- 中央ペイン: プレビュー、再生、タイムライン同期、処理状況
- 右ペイン: タイムライン一覧

最初に行うこと:
1. 上部の入力欄へ動画を指定する
2. 必要なら設定画面で抽出間隔や OCR 条件を調整する
3. `抽出` を押して解析を開始する

### 4.2 設定画面
![設定画面](images/app-settings.png)

主な役割:
- 基本設定: 言語、抽出間隔、出力ディレクトリ
- 検出設定: 前処理、コントラスト、検出候補の確からしさ、最小文字サイズ
- PaddleOCR 設定: `device` と worker 数

使い分けの目安:
- 左上の時計表示など小さい不要文字を無視したい: `最小文字サイズ` を上げる
- 検出漏れが多い: 抽出間隔を短くする、前処理を有効にする
- GPU 環境で試したい: `PaddleOCR device` を `GPU` に切り替える

### 4.3 OCR 実行中の画面
![OCR 実行中のメイン画面](images/app-ocr-running.png)

確認ポイント:
- 処理状況ペインに、現在の進行率、経過時間、残り時間が表示される
- 左ペインの OCR 件数とフレーム数が増えていく
- エラーが起きた場合は `直近の失敗` に原因が出る

長尺動画では、途中で `OCR 再実行` や `出力のみ` を使うより、まず 1 回最後まで通してから結果確認へ進むほうが分かりやすい。

### 4.4 解析結果の確認画面
![字幕検出後の画面](images/app-analysis-result.png)

確認できる内容:
- 右ペインのタイムラインで、時刻、テキスト、表示属性、サイズ、認識精度を一覧できる
- 行を選ぶと、中央プレビューで該当フレームを表示できる
- タイムライン同期や再生を使って、表示位置と前後関係を確認できる
- 誤認識があれば編集し、`出力のみ` で成果物を再生成できる

## 5. おすすめの使い方
### 5.1 まずは短い動画で 1 回確認する
最初から長尺動画を流すより、10 秒から 30 秒程度の短い動画で設定と出力内容を確認するほうが安全である。

確認項目:
- 目的のテロップが拾えているか
- 不要な表示物まで拾っていないか
- `segments.srt` や `segments.csv` が想定用途に合っているか

### 5.2 用途に合わせて出力形式を使い分ける
| 出力ファイル | 向いている用途 |
| --- | --- |
| `segments.srt` | 字幕編集ソフトへ渡す |
| `segments.vtt` | Web 動画向け字幕確認 |
| `segments.ass` | ASS 対応環境での確認用 |
| `segments.csv` | テロップ一覧の確認、レビュー共有 |
| `frames.csv` | フレーム単位の詳細確認 |
| `segments.json` | 他ツール連携、再加工 |

### 5.3 結果確認に時間がかかる場合は `.mtproj` を使う
解析後に `.mtproj` として保存しておくと、次回は動画からやり直さずに同じ状態を開ける。

向いている場面:
- 修正指示を受けて翌日見直す
- 複数人で同じ結果を確認する
- OCR 再実行前の状態を残して比較する

## 6. 利用時の注意
- 実動画 OCR では、`ocr_engine` が `paddleocr` になっていることを確認する。
- 背景や縁取りの影響で、色や表示属性は完全一致ではなく暫定推定になる。
- `font_size` は実フォントサイズではなく、OCR 矩形高さベースの目安である。
- CPU 推論では長尺動画の処理に時間がかかる。まず短い動画で設定確認してから本番動画へ進む。
