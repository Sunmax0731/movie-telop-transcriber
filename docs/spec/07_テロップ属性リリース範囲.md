# テロップ属性リリース範囲

## 1. 目的
初期リリースで保証するテロップ属性の範囲と、既知制約として残す属性を明確にする。

## 2. 初期リリースの判断
初期リリースでは、OCR 結果の確認、編集、字幕出力を優先し、属性推定は補助情報として扱う。

| 項目 | 初期リリースでの扱い | 出力先 | 備考 |
| --- | --- | --- | --- |
| `text` | 対応 | JSON / CSV / SRT / VTT / ASS | OCR と GUI 編集後の代表文字列 |
| `start_timestamp_ms` / `end_timestamp_ms` | 対応 | JSON / CSV / SRT / VTT / ASS | セグメント統合結果の開始/終了時刻 |
| `confidence` | 対応 | JSON / CSV / GUI | OCR confidence の代表値。未取得時は `null` |
| `source_frame_count` | 対応 | JSON / GUI | 統合元フレーム数 |
| `font_size` / `font_size_unit` | 対応。ただし実フォントサイズではない | JSON / CSV / GUI | OCR bounding box の高さ相当値。単位は `px` |
| `text_color` | 暫定対応 | JSON / CSV / GUI | `白文字`、`黒文字`、`緑文字`、`黄文字`、`赤文字`、`青文字` などの分類ラベル |
| `stroke_color` | 暫定対応 | JSON / CSV / GUI | `黒枠`、`白枠` などの分類ラベル |
| `text_type` | 暫定対応 | JSON / CSV / GUI | 色/枠ラベルの結合、または `未分類` |
| `font_family` | 既知制約 | JSON / CSV | 現行実装では `null` |
| `background_color` | 既知制約 | JSON / CSV | 現行実装では `null` |

## 3. 属性値の意味
### 3.1 `font_size`
- OCR bounding box の上下差から算出した高さ相当値である。
- 動画内で実際に使われているフォントポイント数ではない。
- 同一ラン内での相対的な文字サイズ比較には使えるが、デザイン上のフォントサイズとして扱わない。

### 3.2 `text_color` / `stroke_color`
- 現行実装では `#RRGGBB` の厳密な色値ではなく、OCR bounding box 周辺の色分布から推定した分類ラベルである。
- 候補は `白文字`、`黒文字`、`緑文字`、`黄文字`、`赤文字`、`青文字`、`黒枠`、`白枠` を基本とする。
- 低コントラスト、グラデーション、影、複雑な背景では誤分類する場合がある。

### 3.3 `text_type`
- 初期リリースでは「見出し」「人物名」「通常テロップ」などの意味分類ではない。
- `text_color` と `stroke_color` のラベルを `白文字 / 黒枠` のように結合した表示属性ラベルとして扱う。
- 推定できない場合は `未分類` とする。

### 3.4 `font_family` / `background_color`
- 現行実装では推定しない。
- 出力スキーマには将来拡張用に残し、値は `null` または CSV の空欄として扱う。

## 4. 字幕出力での扱い
- SRT / VTT はテキストと時刻のみを使う。
- ASS は初期標準スタイル `Default` のみを使う。
- `text_color`、`stroke_color`、`font_family`、`font_size`、`background_color` を ASS スタイルへ反映しない。
- 属性に基づく ASS スタイル反映は、色値とフォント推定の信頼性を改善した後続タスクで扱う。

## 5. 後続実装候補
| 優先度 | 項目 | 目的 |
| --- | --- | --- |
| P1 | 正確な文字色/枠色の代表色抽出 | ASS スタイル反映と見た目分類の精度向上 |
| P1 | 背景帯/地色の検出 | 帯付きテロップや低コントラストテロップの分類 |
| P2 | `text_type` の意味分類 | 見出し、人物名、通常テロップなどの用途別分類 |
| P2 | ASS スタイル反映 | 字幕ツールで見た目を近づける出力 |
| P3 | フォント名推定 | 厳密なフォント同定は難度が高いため後回し |

## 6. リリース文書への反映
- 導入手順書では、属性推定が補助情報であり、フォント名と背景色は初期リリースでは推定しないことを記載する。
- リリースノートでは、`font_size` が OCR 矩形高さ相当であること、ASS は標準スタイルのみであることを既知制約として記載する。
- QCDS 評価では、文字列と時刻を主要品質指標とし、属性分類は手動確認項目として扱う。
