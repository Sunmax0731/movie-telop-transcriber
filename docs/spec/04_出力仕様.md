# 出力仕様

## 1. JSON 出力スキーマ案
### 1.1 基本方針
- JSON は機械可読な完全出力とする。
- フレーム単位情報とセグメント単位情報を同一ファイルで保持できる構成とする。
- 将来拡張のため、トップレベルに `schema_version` を必須とする。
- 不明値は `null` を使用し、未定義項目の省略は原則行わない。

### 1.2 トップレベル構造
- `schema_version`
- `source_video`
- `processing_settings`
- `frames`
- `segments`
- `run_metadata`

### 1.3 トップレベル項目定義
- `schema_version`
  - 型: `string`
  - 例: `1.0.0`
- `source_video`
  - 型: `object`
  - 内容: 入力動画のメタデータ
- `processing_settings`
  - 型: `object`
  - 内容: 抽出設定、OCR 設定、実行条件
- `frames`
  - 型: `array`
  - 内容: フレーム単位解析結果
- `segments`
  - 型: `array`
  - 内容: 統合済みテロップ単位解析結果
- `run_metadata`
  - 型: `object`
  - 内容: 実行ログ要約、生成日時、アプリバージョンなど

### 1.4 配列要素の扱い
- `frames` は抽出時刻順に並べる。
- `segments` は `start_timestamp_ms` 昇順に並べる。
- 空データの場合も配列自体は出力する。
- 初期実装では `work/runs/<run_id>/output/segments.json` として保存する。

```json
{
  "schema_version": "1.0.0",
  "source_video": {
    "file_path": "D:/videos/sample.mp4",
    "file_name": "sample.mp4",
    "duration_ms": 120000,
    "width": 1920,
    "height": 1080,
    "fps": 29.97
  },
  "processing_settings": {
    "frame_interval_seconds": 1.0,
    "ocr_engine": "paddleocr",
    "offline_mode": true
  },
  "frames": [
    {
      "frame_index": 30,
      "timestamp_ms": 1000,
      "image_path": "work/frames/frame_000030.png",
      "detections": [
        {
          "text": "サンプルテロップ",
          "font_family": null,
          "font_size": 42,
          "font_size_unit": "px",
          "text_color": "白文字",
          "stroke_color": "黒枠",
          "background_color": null,
          "confidence": 0.94
        }
      ]
    }
  ],
  "segments": [
    {
      "segment_id": "seg-0001",
      "start_timestamp_ms": 1000,
      "end_timestamp_ms": 4000,
      "text": "サンプルテロップ",
      "text_type": "白文字 / 黒枠",
      "font_family": null,
      "font_size": 42,
      "font_size_unit": "px",
      "text_color": "白文字",
      "stroke_color": "黒枠",
      "background_color": null,
      "confidence": 0.94
    }
  ],
  "run_metadata": {
    "generated_at": "2026-04-28T07:00:00+09:00",
    "application_version": "0.1.0",
    "processing_time_ms": 18250
  }
}
```

### 1.5 `run_metadata` 項目案
- `generated_at`
  - 型: `string`
  - 形式: ISO 8601
- `application_version`
  - 型: `string`
- `processing_time_ms`
  - 型: `integer | null`
- `warning_count`
  - 型: `integer | null`
- `error_count`
  - 型: `integer | null`

### 1.6 スキーマバージョン方針
- 初期バージョンは `1.0.0` とする。
- 後方互換を壊さない項目追加はマイナー更新とする。
- 既存項目の意味変更や削除はメジャー更新とする。

## 2. CSV 出力スキーマ案
### 2.1 基本方針
- CSV は人間が表計算ソフトで確認しやすい派生出力とする。
- 標準出力はセグメント単位 CSV とする。
- 必要に応じてフレーム単位 CSV を別ファイルで出力する。
- 1 ファイルに異なる粒度のデータを混在させない。

### 2.2 セグメント出力
標準の CSV 出力は、統合済みテロップ単位を 1 行とする。

列案:
- `segment_id`
- `start_timestamp_ms`
- `end_timestamp_ms`
- `text`
- `text_type`
- `font_family`
- `font_size`
- `text_color`
- `stroke_color`
- `background_color`
- `confidence`

列順方針:
- 時系列確認しやすいよう、識別子、時刻、文字列、属性、信頼度の順に固定する。

### 2.3 フレーム単位出力
必要に応じて別 CSV として出力する。

列案:
- `frame_index`
- `timestamp_ms`
- `text`
- `font_family`
- `font_size`
- `text_color`
- `stroke_color`
- `background_color`
- `confidence`

### 2.4 CSV ファイル名案
- セグメント単位: `segments.csv`
- フレーム単位: `frames.csv`

初期実装では、どちらも `work/runs/<run_id>/output/` に保存する。

### 2.5 CSV 出力ルール
- 文字コードは UTF-8 とする。
- 1 行目はヘッダー行とする。
- 不明値は空文字ではなく空欄出力とする。
- 初期リリースの色/枠属性は JSON と同じく `白文字`、`黒枠` などの分類ラベルとして出力する。厳密な `#RRGGBB` 値は後続拡張で扱う。
- 改行を含む OCR 結果は、CSV 上では改行をスペースに正規化してもよい。

### 2.6 JSON と CSV の役割分担
- JSON を完全データとして扱う。
- CSV は確認・集計用の簡易出力とする。
- JSON にある補助情報すべてを CSV に載せることは必須としない。

## 3. 字幕出力
### 3.1 基本方針
- SRT / VTT / ASS は `segments` から生成する字幕ツール連携用の派生出力とする。
- 文字コードは UTF-8 no BOM とする。
- 改行は Windows 配布環境に合わせて CRLF とする。
- 空白のみのテロップは出力しない。
- `end_timestamp_ms` が `start_timestamp_ms` 以下の場合は、字幕ファイルでは終了時刻を開始時刻 + 1000ms として出力する。

### 3.2 ファイル名
- SRT: `segments.srt`
- VTT: `segments.vtt`
- ASS: `segments.ass`

### 3.3 時刻フォーマット
- SRT: `HH:MM:SS,mmm`
- VTT: `HH:MM:SS.mmm`
- ASS: `H:MM:SS.cc`

### 3.4 ASS 初期対応範囲
- ASS は標準スタイル `Default` のみで出力する。
- 初期スタイルは Arial 48px 相当、白文字、黒アウトライン、下中央配置とする。
- 初期リリースでは、`text_color`、`stroke_color`、`font_family`、`font_size`、`background_color` を ASS スタイルへ反映しない。
- テロップ属性に基づく詳細スタイル反映は、正確な代表色抽出とフォント推定の後続実装で扱う。

## 4. 初期実装の出力範囲
- `segments.json`: 入力動画、処理設定、フレーム単位検出、セグメント単位結果、実行メタデータを含む。
- `segments.csv`: セグメント単位の確認用 CSV。
- `frames.csv`: フレーム単位の確認用 CSV。
- `segments.srt`: 一般的な字幕編集ツール向けの SRT。
- `segments.vtt`: WebVTT 対応ツール向けの VTT。
- `segments.ass`: 初期標準スタイル付きの ASS。
- `logs/run.log`: ラン単位の人間向け実行ログ。
- `logs/summary.json`: ラン単位の機械可読な実行要約。
- OCR 検出が 0 件の場合も、JSON、CSV、SRT、VTT、ASS は生成する。
- GUI の設定ボタンは、非モーダル設定画面を開き、言語、抽出間隔、出力先フォルダ、検出設定を扱う。出力形式は設定画面の現在設定サマリで確認でき、各ファイルパスは出力フォルダ配下のランディレクトリで確認する。
- メイン画面左ペインでは、状態カードとして出力フォルダのみを表示し、コピー操作を提供する。
- 出力先フォルダは解析開始前に絶対パスへ正規化し、存在しない場合は作成する。書き込みテストに失敗した場合は `OUTPUT_ROOT_UNAVAILABLE` として処理を開始しない。
- タイムライン上で編集、削除したテロップは、現在の画面内結果へ反映する。JSON / CSV へ反映する場合は、`出力のみ` を実行して現在結果から再出力する。結合 / 分割は再設計まで UI から非表示とする。
- `segments.json` には `edits` 配列を含める。各要素は `operation`、`target_id`、`related_id`、`detection_id`、`original_text`、`updated_text`、`edited_at`、`notes` を持ち、GUI 操作後の出力が元 OCR / セグメント結果からどう変更されたかを追跡できるようにする。
