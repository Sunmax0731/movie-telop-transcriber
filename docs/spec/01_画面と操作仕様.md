# 画面と操作仕様

## 1. 対象
メイン画面構成、サブ画面方針、タイムライン表示、主要操作を定義する。

## 2. 仕様方針
- Windows 上で動作する GUI アプリとする。
- 主画面 1 画面で主要操作を完結させる。
- 結果確認、設定、詳細表示は必要に応じてサブ画面で提供する。
- サブ画面はメイン画面をブロックしない非モーダルダイアログ相当の扱いとする。
- 主要機能はオフライン環境で動作する。

## 3. 採用方針
- Windows 向けデザインシステムは Fluent 2 系を前提とする。
- 実装フレームワーク候補は WinUI 3 を第一候補とする。
- 標準コントロール、タイトルバー、ダイアログ、配色、アクセント挙動は Windows の標準体験に寄せる。
- メイン画面は WinUI 3 の標準レイアウトコンポーネントを中心に構成する。
- 独自装飾を優先せず、可読性と密度を重視する。

## 4. 画面構成
メイン画面 1 画面で以下を縦または左右分割で配置する。
- 動画入力エリア
- 解析設定エリア
- 処理実行エリア
- 進捗表示エリア
- タイムライン表示エリア
- 抽出結果一覧エリア
- プレビューエリア

## 5. メイン画面レイアウト案
初期レイアウトは 3 ペイン構成を基本とする。
- 左ペイン
  - 動画入力
  - 解析設定
  - 実行操作
- 中央ペイン
  - 動画プレビュー
  - タイムライン表示
  - 現在フレーム情報
- 右ペイン
  - 抽出結果一覧
  - 選択中テロップ詳細

補足:
- 左ペインと右ペインは左右の視覚バランスを優先して同じ幅にする。
- 現行実装では右ペインに合わせて左ペインも 340px とする。
- トップ領域ではアプリ名と概要文を表示せず、動画パス入力、参照、抽出、設定と出力の操作を横並びにする。
- 横幅不足時は、左ペインと右ペインを折りたたみ可能にする。
- 主要操作ボタンは左上から左中段に集約し、視線移動を短くする。

## 6. サブ画面方針
- 設定と出力
- エラー詳細
- 属性詳細

これらは非モーダルで開き、表示中もメイン画面を操作可能とする。

サブ画面の振る舞い:
- 同時に複数開ける構成を許容する。
- メイン画面の操作をブロックしない。
- 直前の表示位置とサイズを保存できる余地を持たせる。

初期実装では、設定と出力を 1 つの WinUI `Window` として実装する。この画面はメイン画面と同じ `MainPageViewModel` を参照し、抽出間隔、言語設定、OCR エンジン、出力パス、結果一覧を同じ状態から表示する。再オープン時は既存ウィンドウが残っていれば再利用し、閉じられていれば新規作成する。

## 7. メイン画面の主な操作
- 動画ファイル選択
- 抽出間隔設定
- 言語設定
- 設定と出力画面の表示
- 解析開始
- 解析停止
- 結果保存
- フレーム抽出からの再実行
- OCR からの再実行
- 出力のみの再実行
- タイムライン位置の選択
- タイムライン上のテロップ区間選択
- 結果一覧の選択
- 選択結果に対応する時刻プレビュー

## 8. メイン画面の主要表示項目
- 入力動画の基本情報
  - ファイル名
  - 長さ
  - 解像度
  - フレームレート
- 現在の処理状態
  - 待機中
  - 実行中
  - 完了
  - エラー
- 直近の失敗状態
  - 失敗段階
  - エラーコード
  - エラーメッセージ
  - 関連ログパス
- 出力、ログ、作業ディレクトリ
  - 項目名とパスは横並びにせず、項目名の次の行にパスを表示する。
- 項目説明
  - 項目名の下に常時表示せず、項目領域または入力欄のツールチップとして表示する。
- 現在選択中の結果情報
  - 時刻
  - テキスト
  - 種別
  - 色属性
  - 信頼度
- タイムライン上のセグメント表示
  - 表示開始時刻
  - 表示終了時刻
  - テロップ文字列
  - 選択状態

## 9. 画面操作の基本原則
- 主要操作はメイン画面上で完結させる。
- 頻繁に使う設定はメイン画面に常設する。
- 言語設定は OS の UI 言語を初期値とし、日本語、英語、中国語、韓国語を選択肢として提供する。
- OS の UI 言語が初期対応外の場合は英語を選択する。
- 言語変更時は、主要な固定ラベル、操作ボタン、設定サマリ、Status 項目名を選択言語へ更新する。
- ボタンは表示文字列が見切れない幅または配置にする。右ペイン下部の再実行操作は縦並びの全幅ボタンを基本とする。
- 詳細設定や補助情報のみをサブ画面へ逃がす。
- 実行中も結果一覧の参照や既存結果確認ができる構成を優先する。
- タイムライン、結果一覧、プレビューは選択状態を相互連動させる。

## 10. タイムライン表示仕様
- 動画読込後に動画長さに対応したタイムラインを表示する。
- テロップ統合結果はセグメント単位でタイムライン上へ表示する。
- タイムライン上の各セグメントは少なくとも以下と紐づく。
  - 開始時刻
  - 終了時刻
  - テロップ文字列
- 同一時刻に複数セグメントが重なる場合は、視認可能な重なり表現または上下分離表示を行う余地を持たせる。
- タイムライン選択時は、対応するセグメントを結果一覧とプレビューへ反映する。
- 結果一覧でセグメント選択した場合は、タイムライン側も該当区間をハイライトする。
- プレビュー時刻移動時は、該当時刻をタイムライン上にカーソル表示できる構成を優先する。

## 11. UI 方針
- 入力欄、設定欄、一覧、ボタン、通知は WinUI 3 標準コントロールを優先する。
- 情報のまとまりはカード乱用ではなく、グループ化と余白で整理する。
- 状態表示には `InfoBar` 相当の通知 UI を優先する。
- 一覧表示にはソートや列拡張余地のあるテーブル系 UI を想定する。
- テーマは Windows のライト / ダーク切替に追従できる構成を優先する。
- タイムライン表示は標準コントロールの組み合わせまたは軽量カスタム描画で実現する。

## 12. 設計工程への引き継ぎ事項
- ペインの最小幅と折りたたみ挙動
- プレビュー領域の再生操作有無
- 結果一覧の列定義
- サブ画面の永続化対象
- タイムラインのズーム、スクロール、重なり表現
