# QCDS評価仕様

## 1. 目的
UI/UX と OCR 改善の成果を主観だけで判断せず、代表動画セットと計測レポートで継続比較できる状態にする。

## 2. QCDS の扱い
本リポジトリでは、リリース前評価の QCDS を以下の意味で扱う。

| 区分 | 評価対象 | 主な指標 |
| --- | --- | --- |
| Q | OCR とセグメント品質 | 文字列完全一致率、文字単位一致率、欠落、余計な検出 |
| C | 処理コスト | 処理時間、フレーム数、OCR 検出数、依存ランタイム |
| D | 時刻追跡と出力成立 | セグメント開始/終了時刻誤差、成果物生成、ログ生成 |
| S | 保守性と利用継続性 | エラー件数、警告件数、手動確認結果、既知制約 |

## 3. 代表動画条件
リリース前の継続評価では、最低限以下の条件を代表動画セットに含める。

| 条件 | 目的 | 自動評価 | 手動確認 |
| --- | --- | --- | --- |
| 日本語横書きテロップ | 初期対象の主要ケースを確認する | 文字列一致、時刻誤差 | 読みやすさ、プレビュー同期 |
| 小書き仮名を含むテロップ | 拗音、促音、撥音の誤認識を確認する | 文字単位一致、置換傾向 | 不自然な通常文字化 |
| 低コントラストテロップ | 背景と文字色が近いケースを確認する | 欠落、confidence、余計な検出 | 視認性、矩形ずれ |
| 複数位置のテロップ | 上部、中央、下部の検出順と同期を確認する | 時刻誤差、余計な検出 | 上から順の表示、プレビュー枠 |
| 短時間表示テロップ | 抽出間隔とセグメント統合の限界を確認する | 欠落、開始/終了時刻誤差 | 実利用上の見落とし |

初期レポートは `test-data/basic_telop/sample_basic_telop.mp4` を使う。このサンプルは日本語横書き、表示時刻の追跡、短時間セグメント、装飾差分を最小構成で確認する。小書き仮名、低コントラスト、複数位置の実動画評価は、追加サンプルまたは権利確認済み動画のレポートで拡張する。

## 4. 計測項目
自動評価で扱う指標は以下とする。

| 指標 | 算出方法 | 目的 |
| --- | --- | --- |
| 認識文字列完全一致率 | 正解セグメント数に対する、正規化後テキスト完全一致数 | テロップ文字列としてそのまま使える割合を見る |
| 文字単位一致率 | Levenshtein 距離から算出した一致率の平均 | 誤字、脱字、余計な文字の傾向を見る |
| 欠落セグメント数 | 正解に対してマッチしなかったセグメント数 | 未検出リスクを見る |
| 余計な検出セグメント数 | 解析結果に残った未マッチセグメント数 | 背景文字や時刻表示などの混入を見る |
| 開始/終了時刻誤差 | マッチしたセグメントの開始/終了時刻差分 | 字幕出力とプレビュー同期の精度を見る |
| 処理時間 | `run_metadata.processing_time_ms` または `summary.json` の開始/完了時刻差 | 実用速度を見る |
| 警告/エラー件数 | `summary.json` と `segments.json` の run metadata | 安定性と導入リスクを見る |

## 5. 自動評価と手動確認の分離
自動評価対象:
- 正解 JSON と `segments.json` の比較。
- 件数、文字列、時刻、処理時間、警告/エラーの集計。
- 改善前後の差分比較。

手動確認対象:
- プレビュー overlay の位置ずれ。
- タイムライン上の選択行とプレビュー枠の同期。
- テロップ種別ラベルの妥当性。
- GUI 操作の分かりやすさ。
- 実動画に対する見落としや許容できない余計な検出。

## 7. レポート運用
- 代表動画ごとに 1 レポートを作成する。
- 同じ動画で設定や実装を変えた場合は、前回メトリクス JSON を `--previous-metrics` に指定して差分を残す。
- 実動画の権利や容量に制約がある場合、動画ファイルはコミットせず、レポートにメタ情報、Run ID、評価コマンドを残す。
- QCDS 指標が悪化した場合は、原因が OCR、セグメント統合、属性解析、GUI 操作のどこにあるかを切り分けて Issue 化する。
