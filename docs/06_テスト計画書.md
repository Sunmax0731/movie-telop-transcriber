## 3. テスト範囲
対象:
- WinUI 3 アプリの起動と主要画面表示
- 動画ファイル選択とメタデータ表示
- フレーム抽出と `work/runs/<run_id>/frames/` への保存

- `logs/run.log`、`logs/summary.json` の出力
- 設定ボタンから開く、詳細設定用の非モーダル子ウィンドウ表示
- フレーム抽出、OCR、出力の段階別再実行
- エラー表示と直近ログパス表示
- オフライン導入を想定した起動と実行

対象外:
- 実動画に対する OCR 精度の最終評価
- PaddleOCR / Tesseract / Windows 標準 OCR の最終採用判断
- 配布物作成と GitHub Release

## 6. テスト種別
### 6.1 ビルド確認
- ソリューションが復元、ビルドできること。
- 警告とエラーを記録すること。

判定基準:
- Debug x64 ビルドが成功する。
- 既知制約がない限り、エラーは 0 件であること。

### 6.2 機能テスト
- 動画選択後にファイル名、長さ、解像度、FPS、Codec が表示されること。
- 抽出間隔を変更してフレーム抽出数に反映されること。
- ランごとに `work/runs/<run_id>/` が作成されること。
- フレーム、OCR、属性、出力、ログが所定のディレクトリへ保存されること。

判定基準:
- 各成果物のパスが GUI とファイルシステムで一致する。
- ランが分離され、前回実行の成果物を上書きしない。

### 6.3 GUI 操作テスト
- メイン画面で動画選択、解析開始、タイムライン選択、プレビュー確認ができること。
- 設定ボタンから、詳細設定用の子ウィンドウを開けること。
- サブ画面表示中もメイン画面を操作できること。
- 設定画面の言語、出力先フォルダ、PaddleOCR 調整がメイン画面と処理設定へ反映されること。
- 設定画面に、メイン画面と重複する最新結果一覧や個別出力パス操作が表示されないこと。

判定基準:
- 操作不能、クラッシュ、明らかな表示不整合がないこと。

### 6.4 OCR / 属性解析テスト
- `.ocr.json` サイドカーを配置したフレームで OCR レスポンスが読み込まれること。
- テキスト、信頼度、矩形情報が内部モデルへ格納されること。
- 矩形情報がある場合、フォントサイズ相当値が算出されること。
- 不明な色属性やフォント名は `null` として扱われること。

判定基準:
- サイドカーの入力値が JSON 出力と GUI 表示へ追跡できる。
- 不明値が空文字や欠落ではなく、仕様どおりに扱われる。

### 6.5 セグメント統合テスト
- 同一文字列・同一属性キーの連続フレームが 1 セグメントに統合されること。
- セグメントの開始時刻と終了時刻がフレーム時刻と抽出間隔から算出されること。
- 同一時刻に複数検出がある場合も一覧に表示されること。

判定基準:
- `segments.json` と GUI タイムラインの件数、時刻、文字列が一致する。

### 6.6 出力テスト
- `segments.json` が `schema_version`、`source_video`、`processing_settings`、`frames`、`segments`、`run_metadata` を含むこと。
- `segments.csv` と `frames.csv` が UTF-8 でヘッダー付き出力されること。
- `segments.srt`、`segments.vtt`、`segments.ass` が UTF-8 no BOM、CRLF 改行で生成されること。
- OCR 検出が 0 件の場合も空配列、ヘッダー付き CSV、空またはヘッダーのみの字幕ファイルが生成されること。

判定基準:
- JSON がパース可能であること。
- CSV の列順が仕様と一致すること。

### 6.7 ログテスト
- `logs/run.log` と `logs/summary.json` が生成されること。
- `run_id`、入力動画、抽出間隔、OCR エンジン、件数、成果物パスが記録されること。
- GUI からログパスを確認できること。

判定基準:
- ログと実際の成果物パスが一致すること。

### 6.8 エラーと再実行テスト
- 無効な動画パスで入力検証エラーが表示されること。
- 前段成果物なしで `Rerun OCR` または `Export only` を押した場合、前段不足が表示されること。
- `Rerun frames` はフレーム抽出から後段まで再実行すること。
- `Rerun OCR` は直近フレームを使って OCR 以降を再実行すること。
- `Export only` は直近解析結果を使って出力とログ生成だけを再実行すること。

判定基準:
- `Last failure` に失敗段階、エラーコード、エラーメッセージが表示される。
- 再実行後の成果物が更新される。

### 6.9 オフライン導入テスト
- ネットワーク接続なしでも起動できること。
- 必要ランタイムと同梱物が整理されていること。
- サンプル動画で解析から出力まで実行できること。
- self-contained publish 出力と Release build 出力のどちらを配布候補にできるかを切り分けること。

判定基準:
- 導入手順に不足がないこと。
- 起動と主要機能がオフライン前提で成立すること。

## 12. #95 タイムライン操作と検出設定 UI 追加改善検証
- 日本語を含む Desktop 配下の出力先で、PaddleOCR worker が `UnicodeDecodeError` を出さずに OCR を実行できること。
- `○`、`〇`、`◯`、正方形に近い単独の `O` / `0` が `◎` として扱われること。
- PaddleOCR が空文字を返した二重丸形状の検出領域が `◎` として補完されること。
- 処理状況に経過時間と推定残り時間が表示され、実行中は 1 秒間隔で更新されること。
- 言語設定を切り替えると、経過時間、残り時間、フレーム単位などの進捗ラベルが切り替わること。
- 設定画面の固定サイズが `1680 x 720` で、リサイズと最大化ができないこと。
- 設定画面上部にタイトルと説明文が表示されず、左ペインの見出しが基本設定になっていること。
- 設定画面の PaddleOCR 前処理、検出設定、ツールチップが言語設定に応じて切り替わること。
- タイムライン詳細列の見出しが表示内容に合い、confidence や出現フレーム数が重複表示されないこと。
- タイムライン詳細列の内容が同じ値で二行表示されず、一行表示になること。
- タイムラインのフレーム列が数値のみで表示されること。
- タイムライン選択時にコピー、編集、削除ボタンが同一幅で表示され、コピーでテキストがクリップボードへ保存されること。
- タイムライン見出し右クリックで列表示 ON/OFF を切り替えられ、最後の 1 列は非表示にできないこと。
- タイムライン種類ラベルに `白文字`、`緑文字`、`黒枠`、`未分類` などが表示されること。
- プレビューのシーケンスバーで表示フレームを変更でき、該当タイムライン行があれば選択も同期すること。
- 設定画面の抽出間隔、コントラスト、検出しきい値、検出ボックスしきい値、unclip ratio、limit side length をスライダーで変更できること。
- 初期設定へ戻すボタンで、言語、抽出間隔、PaddleOCR 前処理、検出設定が既定値へ戻ること。
- 処理状況ペイン内で、再実行ボタン 3 つが横並びで表示されること。
- 直近の失敗が処理状況ペインではなく、左ペインの状態セクションに表示されること。
- 処理状況ペインの高さが従来より抑えられ、中央プレビュー領域を圧迫しないこと。

## 16. #102 OCR エンジン未設定時の誤起動防止検証
- OCR エンジンと外部 worker を未指定で起動した場合、アプリの OCR エンジン表示が `paddleocr` になること。
- PaddleOCR を導入していない Python が使われた場合、OCR 未構成が OCR エラーとして表示され、`json-sidecar` の空検出成功として完了しないこと。
- `MOVIE_TELOP_OCR_ENGINE=json-sidecar` を明示し、対象フレームの `.ocr.json` が存在しない状態で OCR を実行した場合、`OCR_SIDECAR_NOT_FOUND` が直近失敗と OCR response JSON に記録されること。
- `MOVIE_TELOP_OCR_ENGINE=json-sidecar` を明示し、対象フレームの `.ocr.json` が存在する状態では、従来どおり sidecar の OCR 結果を読み込めること。
- `MOVIE_TELOP_OCR_WORKER` を指定した場合は、外部 JSON worker 経路が維持されること。

- OCR 実行後、中央プレビューに選択中フレーム画像が表示されること。
